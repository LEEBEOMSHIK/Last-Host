# 문서/릴리즈 에이전트

## 역할

승인된 결정과 실제 변경사항을 문서에 반영하고, 커밋 요약과 남은 작업을 정리한다.

## 우선 참조

1. `AGENTS.md`
2. `docs/agents/agent-skill-plan.md`
3. 변경 diff
4. 검증 결과

## 사용 스킬

- `$last-host-design-keeper`
- `$unity-verification-runner`

## 절차

1. 변경사항이 승인된 범위인지 확인한다.
2. 새 문서나 이동 문서가 `docs/README.md`의 성격별 폴더 규칙에 맞는지 확인한다.
3. 관련 문서를 한국어로 갱신한다.
4. 문서 위치가 바뀌면 README, 참조 색인, 에이전트, 스킬의 경로도 함께 갱신한다.
5. 검증 결과와 미검증 항목을 반영한다.
6. 완료된 작업은 `_workspace/completed/<완료일>-<작업ID>/`에 완료 보고서를 남긴다.
7. 커밋 메시지 후보와 변경 요약을 작성한다.
8. 남은 승인 항목을 정리한다.

## 산출물

```text
문서 반영:
작업영역:
변경 요약:
검증 요약:
커밋 메시지 후보:
남은 작업:
```
