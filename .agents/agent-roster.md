# 에이전트 역할 목록

이 문서는 `마지막 숙주 / The Last Host` 프로젝트에서 사용할 에이전트 역할을 빠르게 참조하기 위한 목록이다. 상세 운영 절차는 `docs/agents/agent-skill-plan.md`를 따른다.

## 공통 규칙

- 모든 에이전트는 `AGENTS.md`를 우선한다.
- 모든 사용자-facing 산출물은 한국어로 작성한다.
- 승인 게이트에 걸리는 작업은 제안만 하고 실행하지 않는다.
- 프로젝트 방향, 범위, 승인 게이트에 영향을 주는 주요 산출물은 프로젝트 총괄 관리자 에이전트 검토 후 사용자에게 올린다.
- 에이전트 결과물은 조정자 검토 후 반영한다.
- 작업 배정과 진행 기록은 `_workspace/active/<작업ID>/`에 남긴다.
- 완료된 작업은 `_workspace/completed/<완료일>-<작업ID>/`에 완료 보고와 검증 기록을 남긴다.
- 스킬 사용은 에이전트 위임을 대체하지 않는다.
- 코드, 씬, ProjectSettings, 승인 문서, 운영 문서 변경은 완료 또는 커밋 보고 전 QA/검증 에이전트 기록과 프로젝트 총괄 관리자 판정이 있어야 한다.
- 코드, 씬, 테스트, ProjectSettings 변경은 조정자가 직접 수행하지 않고 해당 구현 에이전트에 배정한다. 예외는 사용자 명시 승인과 작업 이력의 예외 사유가 있을 때만 허용한다.

## 1. 프로젝트 총괄 관리자 에이전트

- 파일: `.agents/project-director-agent.md`
- 목적: 사용자 최종 승인 전 내부 승인, 프로젝트 방향/범위/승인 게이트 점검
- 주요 입력: 사용자 요청, 조정자 통합 보고, 에이전트 산출물, `AGENTS.md`, `docs/`
- 주요 산출물: 내부 승인 판정, 수정 필요 항목, 사용자 결정 필요 항목

## 2. 프로젝트 조정 에이전트

- 파일: `.agents/project-coordinator-agent.md`
- 목적: 작업 분해, 담당 배정, 결과 통합, 승인 질문 정리
- 주요 입력: 사용자 요청, `AGENTS.md`, `docs/`
- 주요 산출물: 작업 배정안, 승인 필요 항목, 최종 통합 보고

## 3. 기획 정리 에이전트

- 파일: `.agents/design-keeper-agent.md`
- 목적: 원본 기획과 작업 문서의 일관성 유지
- 주요 입력: 원본 `.docx`, `docs/design/game-design-summary.md`, `docs/prototype/official/rat-host-prototype.md`
- 주요 산출물: 기획 요약, 범위 변경 감지, 승인 질문

## 4. Unity 아키텍처 에이전트

- 파일: `.agents/unity-architecture-agent.md`
- 목적: Unity 버전, URP, 폴더 구조, 씬 구조, 시스템 경계 계획
- 주요 입력: `docs/project/project-prep.md`, 프로토타입 범위 문서
- 주요 산출물: Unity 구조안, 패키지/설정 승인 목록

## 5. 게임플레이 루프 에이전트

- 파일: `.agents/gameplay-loop-agent.md`
- 목적: 쥐 숙주 조종, 면역 경계도, 내부 미니게임, 변이 선택 루프 설계
- 주요 입력: `docs/prototype/official/rat-host-prototype.md`, Unity 구조안
- 주요 산출물: 기능 단위 작업 목록, 수용 기준, 테스트 시나리오

## 6. 비주얼/테크아트 에이전트

- 파일: `.agents/visual-tech-art-agent.md`
- 목적: 도트풍 하이브리드 2.5D 비주얼, 3D 환경, 3D 캐릭터 원본, 8방향 프리렌더드 스프라이트, 카메라·렌더링·플레이스홀더 기준 관리
- 주요 입력: `docs/design/game-design-summary.md`, 비주얼 승인사항
- 주요 산출물: 스타일 기준, 원본·방향 렌더·피벗·가림 기준, 렌더링 기준, 에셋 목록

## 7. QA/검증 에이전트

- 파일: `.agents/qa-verification-agent.md`
- 목적: 완료 전 검증, 테스트/빌드/Unity MCP 플레이 체크 절차 관리
- 주요 입력: 구현 계획, 변경 diff, 테스트 로그
- 주요 산출물: 검증 체크리스트, 테스트 결과, MCP 플레이 체크 결과, 남은 위험
- 역할 경계: 사용자 최종 확인 전 사전 검증을 담당하며, 프로젝트 총괄 관리자 에이전트의 내부 승인 판정은 대신하지 않는다.

## 8. 게임플레이 구현 에이전트

- 파일: `.agents/gameplay-implementation-agent.md`
- 목적: 승인된 게임플레이 C# 코드와 관련 테스트를 실제로 구현
- 주요 입력: 작업 패킷, 구현 계획, 게임플레이 루프 에이전트 산출물, 기존 코드/테스트
- 주요 산출물: C# 코드 변경, EditMode 테스트, 구현 결과 핸드오프

## 9. Unity 씬/통합 구현 에이전트

- 파일: `.agents/unity-scene-integration-agent.md`
- 목적: 승인된 씬, 프리팹, 입력, 카메라, UI, 빌드 설정 연결을 실제로 구현
- 주요 입력: 작업 패킷, Unity 아키텍처 산출물, 비주얼 기준, 구현된 컴포넌트
- 주요 산출물: 씬/프리팹/설정 변경, Unity 연결 결과, Play 확인 기록

## 10. 문서/릴리즈 에이전트

- 파일: `.agents/documentation-release-agent.md`
- 목적: 문서 최신화, 커밋 요약, 마일스톤 정리
- 주요 입력: 승인된 설계, 변경 diff, 검증 결과
- 주요 산출물: 문서 수정안, 커밋 요약, 릴리즈/마일스톤 요약

## 11. Blender 애니메이션 테크아트 에이전트

- 파일: `.agents/blender-animation-tech-artist-agent.md`
- 목적: 승인된 저폴리 캐릭터 Blender 원본, 간단한 리그·보행 애니메이션, 고정 8방향 시험 렌더를 실제 제작·수정
- 주요 입력: 작업 패킷, 비주얼/테크아트 기준, 기존 Blender 원본·렌더 설정
- 주요 산출물: 버전 분리된 `.blend`, 프레임별 8방향 PNG, 프레임 맵·렌더 설정, Blender 검증 핸드오프
- 역할 경계: Unity 반입·최종 비주얼 승인·QA 판정은 담당하지 않음
