# 작업 패킷: 면역 신호 억제 미니게임 구현

## 목표

`면역 신호 억제` 리듬형 내부 미니게임의 1차 구현을 추가한다. 기존 백혈구 회피 미니게임은 기본값으로 유지하고, 새 유형은 별도 내부 미니게임 타입으로 선택 가능하게 한다.

## 구현 범위

- 리듬형 신호 억제 순수 판정 모델 추가
- 내부 미니게임 타입 구분 추가
- 세션 상태에서 신호 억제 성공/실패를 변이 선택/실패 복귀 루프에 연결
- HUD에 신호 억제 진행 상태와 판정 피드백 표시
- 개발/검증용으로 신호 억제 타입 진입 가능하게 연결
- EditMode 테스트 추가

## 비범위

- 기존 백혈구 회피 미니게임 수정
- 음악/사운드 싱크 구현
- 2키 입력, 가짜 신호, 긴 입력 패턴
- 최종 UI/전용 노트 프리팹 제작
- 원인별 미니게임 자동 선택 시스템 완성
- 씬 배치 대규모 변경

## 수용 기준

1. 기본 내부 미니게임 타입은 기존 백혈구 회피로 유지된다.
2. `면역 신호 억제` 타입으로 진입하면 백혈구 조각 수집 대신 신호 억제 모델이 진행된다.
3. 정확 타이밍 입력이 목표 수 이상 누적되면 변이 선택으로 전환된다.
4. 빠름/늦음/누락이 누적되어 안정도가 0이 되거나 남은 신호로 목표 달성이 불가능하면 실패로 전환된다.
5. HUD에서 `면역 신호 억제`, `신호 차단`, `경보 증폭` 상태를 확인할 수 있다.
6. 기존 백혈구 회피 성공/실패 테스트는 회귀하지 않는다.
7. Unity MCP Play 체크가 가능하면 신호 억제 진입과 성공/실패를 확인한다. 불가능하면 사유와 대체 검증을 기록한다.

## 직접 구현 예외

- 사용자 명시 요청으로 구현을 진행한다.
- 실제 병렬 하위 에이전트 실행은 비용 절감을 위해 생략하고, 역할별 산출물과 판정을 작업 기록에 남긴다.
- 새 미니게임 구현은 이번 사용자 지시로 승인된 범위에 한정한다.
