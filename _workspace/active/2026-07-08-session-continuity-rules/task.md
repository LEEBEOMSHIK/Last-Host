# 작업 패킷: 세션 연속성 기준 추가

## 목표

토큰 또는 세션이 끊기기 전에 다른 AI나 다른 세션이 작업을 바로 이어받을 수 있도록 `_workspace`의 인수인계 기준을 보강한다.

## 구현 범위

- `_workspace/README.md`에 세션 연속성 기준과 토큰 경계 규칙 추가
- `_workspace/templates/handoff.md`를 다음 세션용 인수인계 형식으로 보강
- `_workspace/active/CURRENT.md` 전역 포인터 추가
- 이번 기준 추가 작업의 진행/검증/총괄 판정 기록

## 비범위

- 새 에이전트 역할 생성
- 새 Codex 스킬 생성
- AGENTS.md 장문화
- 기존 활성 작업 폴더 정리 또는 완료 처리
- 커밋/푸시

## 수용 기준

1. 다음 세션이 `_workspace/active/CURRENT.md`만 보고 이어받을 작업과 읽을 문서를 알 수 있다.
2. `handoff.md`에는 최신 사용자 요청, 현재 상태, 변경 파일, 검증, 차단 항목, 다음 작업이 남도록 한다.
3. 토큰 경계는 숫자만이 아니라 작업 단계 기준으로 명시된다.
4. 80% 이상 토큰 소모 추정 시 새 구현보다 인수인계를 우선한다는 기준이 문서화된다.
5. 운영 규칙은 AGENTS.md가 아니라 `_workspace` 운영 문서에 둔다.

## 담당 배정

| 담당 | 역할 | 산출물 |
| --- | --- | --- |
| 프로젝트 조정 에이전트 | 범위 정리, 파일 반영 | 작업 패킷, 문서 변경 |
| 문서/릴리즈 에이전트 | 운영 문서 문구 정리 | `_workspace/README.md`, 템플릿 |
| QA/검증 에이전트 | 문서 구조와 링크 확인 | `verification.md` |
| 프로젝트 총괄 관리자 에이전트 | 범위/승인 게이트 판정 | `agent-activity.md` 판정 |
