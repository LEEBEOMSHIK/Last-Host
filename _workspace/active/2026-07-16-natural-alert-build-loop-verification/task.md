## 검증 주장

기존 Windows 빌드 `Builds/RatHostPrototype/LastHostPrototype.exe`에서 디버그 진입이나 런타임 상태 주입 없이 `RatHost 시작 → 자연 면역 경계도 100% → 기본 WhiteBloodCellEvasion → 변이 조각 3개 수집 → 변이 선택 → 선택 변이가 적용된 RatHost 복귀`를 실제 플레이 입력과 화면 관측으로 한 번 이상 완료할 수 있다.

이 문장의 모든 단계가 같은 실행 세션의 연속 증거로 확인될 때만 `엄격 검증 통과`를 주장한다.

## 목적

이전 빌드 성공 루프가 `F6` 디버그 진입의 `면역 신호 억제` 경로였다는 검증 공백을 닫는다. 이번 작업은 자연 면역 경계도 100% 도달과 기본 `WhiteBloodCellEvasion` 조각 수집 성공을 Windows 빌드에서 직접 증명하며, 사용자 수동 조작감·난이도·무설명 이해 여부 보류 항목과는 분리한다.

## 허용 입력과 관측 방식

- 플레이어용 입력인 `WASD`, `Space`, 변이 선택용 숫자 `1`~`3`, 또는 동일한 공개 UI 버튼 클릭.
- 자연 시간 경과와 승인된 게임 내 위험 행동을 이용한 면역 경계도 상승.
- 화면 캡처, 창 제목·포커스 기록, 실행 시각·단계별 관측 메모, 해당 세션 `Player.log` 복사와 오류 패턴 검색.

## 해야 할 일

1. 기존 빌드와 Git 작업 상태를 읽기 전용으로 확인하고, 빌드 재생성 없이 실행한다.
2. 시작 시 RatHost 모드, 면역 경계도, 변이 없음 상태와 게임 창 포커스를 증거로 남긴다.
3. `F6` 없이 자연 시간 경과 또는 게임 내 위험 행동으로 면역 경계도 100%에 도달하고 기본 `WhiteBloodCellEvasion` 전환을 확인한다.
4. 실제 이동 입력으로 백혈구를 피하면서 변이 조각 3개를 수집하고 수량 변화를 단계별 화면으로 확인한다.
5. 변이 선택 화면에서 공개 플레이 입력으로 변이 1개를 선택한다.
6. RatHost 복귀, 선택 변이 표시·효과 상태, 복귀 면역 경계도를 확인한다.
7. 같은 실행 세션의 `Player.log`에서 `Exception`, `Error`, `Crash`와 치명적 실행 중단 징후를 확인한다.
8. 성공·실패·미검증을 분리해 `verification.md`와 `artifacts/`에 기록한다.
9. QA 판정 뒤 프로젝트 총괄 관리자 판정과 상태판 독립 대조를 받는다.
10. 커밋 직전 상태판·`CURRENT.md`·작업 패킷·Git 상태를 동기화하고, 허용 범위만 커밋·푸시한다.

## 성공 증거 최소 세트

- 시작 증거: 대상 창 제목·포커스, RatHost 모드, 시작 면역 경계도, 시작 변이 상태가 한 시도에 연결되는 기록.
- 자연 전환 증거: `F6` 또는 직접 상태 전환 없이 경계도 상승 과정과 100% 도달 직후 기본 `WhiteBloodCellEvasion` 화면.
- 내부 성공 증거: 백혈구 회피 화면과 변이 조각 `0/3`에서 `3/3`까지의 단계별 관측. 최소한 시작, 중간 수량, 완료 수량을 화면으로 남긴다.
- 선택 증거: 세 가지 변이 선택 화면과 실제 선택 입력·선택 대상.
- 복귀 증거: RatHost 모드, 획득 변이 표시, 복귀 면역 경계도 및 선택 변이 적용 상태.
- 로그 증거: 같은 성공 시도의 `Player.log` 원본 또는 사본과 `Exception`·`Error`·`Crash` 검색 결과.
- 연속성 증거: 위 증거가 같은 프로세스/실행 시도임을 시각·시각표 또는 시도 ID로 연결한 요약.

## 실패와 미검증 분리 기준

- 경계도 100% 전에 게임이 종료되거나 다른 내부 미니게임으로 들어가면 해당 시도는 실패다.
- 백혈구 접촉, 안정도 0, 조각 3개 미수집, 실패 패널 복귀는 실패 시도로 기록한다.
- 포커스 증거가 없거나 입력 전달 여부를 화면 상태로 확인할 수 없으면 실제 입력은 미검증이다.
- `F6`, MCP/콘솔 명령, 세션 API, Inspector, 메모리 조작 등으로 모드를 직접 전환한 시도는 이번 주장에 사용할 수 없다.
- 시작·내부·선택·복귀 증거가 서로 다른 세션이면 연속 성공 루프는 미검증이다.
- 로그를 성공 시도와 연결할 수 없거나 오류 검색을 하지 못하면 로그 안전성은 미검증이다.
- 성공 증거가 일부만 있으면 확인된 단계만 통과로 기록하고 전체 엄격 검증은 통과로 승격하지 않는다.
- 실제 사용자 조작감, 난이도, 설명 없이 목표 이해 여부는 이번 대리 검증 범위 밖 미검증으로 계속 보류한다.
