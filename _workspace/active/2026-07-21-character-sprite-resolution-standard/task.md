# 작업 배정서 — 캐릭터 스프라이트 해상도 상향과 공통 기준

## 기본 정보

- 작업 ID: `2026-07-21-character-sprite-resolution-standard`
- 상태: 회귀 QA 통과 / 사용자 화면 확인·EditMode TestRunner·총괄 재검토 대기
- 사용자 승인: 2026-07-21, 현재 쥐 v3 프리렌더 해상도 상향·Unity 시험 반입·향후 오브젝트 공통 기준 문서화
- 담당 에이전트: Blender 애니메이션 테크아트, Unity 씬/통합, QA/검증
- 보조 에이전트: 비주얼/테크아트, 프로젝트 총괄
- 사용 스킬: `$pixel-lowpoly-style-keeper`, `$last-host-design-keeper`, `$unity-verification-runner`

## 목적

현재 64×64 쥐 v3 시험 프레임의 선명도를 높이되 Unity에서의 월드 표시 크기와 3D 게임플레이 루트를 보존한다. 검증 후 향후 프리렌더 캐릭터·게임플레이 오브젝트가 재사용할 공통 출력·Import 기준을 남긴다.

## 확정 작업 기준

- Blender 출력: `128×128` RGBA PNG, 8방향 × 8프레임, 기존 v3 카메라·조명·방향·피벗·월드 화면 점유율 유지
- Unity Import: Sprite Single, PPU `64`, Custom Pivot `(0.5, 0.25)`, Point, 밉맵 끔, 무압축, Clamp
- 표시 크기 보존: 기존 `64px / PPU 32`와 같은 2 Unity unit 캔버스 폭을 유지한다.
- 새 출력은 v4 별도 Blender 원본·별도 Unity 시험 경로만 사용한다.

## 해야 할 일

1. 비주얼/테크아트가 위 기준이 픽셀풍 하이브리드 2.5D와 화면 안정성에 맞는지 검토한다.
2. Blender 담당이 v3를 읽기 전용 입력으로 v4 128px 원본·64 PNG·프레임 맵·렌더 설정을 새 경로에 제작·검증한다.
3. Unity 씬/통합 담당이 v4를 별도 경로로 반입하고 기존 월드 크기를 보존한 채 `RatVisual` 배열을 연결한다.
4. 공통 제작 가이드에 향후 프리렌더 캐릭터·오브젝트 기준과 예외 처리 기준을 기록한다.
5. QA가 Importer·PPU·피벗·정지/이동·8방향·카메라·콘솔·실제 화면을 독립 확인한다.

## 금지 범위

- TrialV1, WalkTrialV2, WalkTrialV3, 기존 Blender 원본의 덮어쓰기·삭제
- 이동·충돌·면역·변이·카메라 조작 로직의 변경
- 패키지, ProjectSettings, Build Settings, URP 구조, 최종 아트 승인
- 새 캐릭터·스테이지·게임 시스템 추가

## 완료 기준

1. v4 64장이 모두 128×128이며 기존 v3와 방향·루프·피벗·접지 기준을 보존한다.
2. Unity v4 Import가 PPU 64와 확정 Import 기준을 충족하고, 기존과 화면 월드 크기가 같다.
3. 정지·이동·8방향·접지·쿼터뷰에서 이미지 교체·흐림·오류가 없는지 QA Play가 기록한다.
4. 제작 가이드에 다음 오브젝트에 적용할 기본 규격·예외·검증 기준이 한국어로 기록된다.
5. QA 기록과 총괄 관리자 판정 전에는 완료·커밋 완료로 주장하지 않는다.
