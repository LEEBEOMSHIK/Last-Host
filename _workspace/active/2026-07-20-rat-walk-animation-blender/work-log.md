# 작업 로그

## 2026-07-20 — Blender 연결 점검

- 사용자 승인과 비주얼 검토를 바탕으로 Blender의 열린 원본을 비파괴적으로 확인하려 했다.
- Windows 앱 조작 연결의 `list_apps`가 초기 호출, 규정된 2초 후 재시도, 세션 초기화 후 재호출까지 모두 native pipe `os error 2`로 실패했다.
- 열린 Blender의 저장 여부, 원본 파일, 씬 오브젝트를 확인하거나 변경하지 않았다.
- Unity·기존 정지 PNG/OBJ·ProjectSettings·패키지 변경은 없다.

## 다음 작업

1. Windows 앱 조작 연결이 복구되면 Blender 창과 저장 상태를 먼저 읽기 전용으로 확인한다.
2. 사용자가 Blender 파일을 저장해 경로를 알려주면, 그 파일을 입력으로 8프레임 걷기 루프 시험 제작을 재개한다.

## 2026-07-20 재개 결과

- 읽기 전용 정지 OBJ를 입력으로 Blender 5.1.2 백그라운드에서 별도 작업본과 8방향 × 8프레임 PNG를 생성했다.
- 자동 QA: PNG 64장, 64×64 RGBA·투명·무AA, 경계 잘림 없음, 방향별 루프 바닥선 고정, f01/f03/f05/f07 위상, root key 0, mesh min Z=0을 확인했다.
- 총괄 재검토: W 방향이 기존 정지 PNG 대비 bbox 하단 1px 위여서 **수정 필요**. UnityProject와 기존 정지 자산에는 쓰기 작업을 수행하지 않았다.

## 2026-07-20 — v3 과장 보행 제작·렌더

- 사용자 피드백(쥐 크기가 아니라 보행 동작 폭을 키워야 함)에 따라, v2를 읽기 전용 입력으로 한 별도 v3 원본을 제작했다.
- 스윙 발의 로컬 X 전진 폭을 `0.40`, 로컬 Z 들림을 `0.28`로 키우고, root가 아닌 메시 셰이프키에만 `0.045` 몸통 리듬과 `0.11` 꼬리 반대 흔들림을 추가했다.
- 실제 Blender `bpy.ops.render.render(write_still=True)`로 `renders-v3/`에 8방향 × 8프레임, 총 64장의 64×64 RGBA PNG를 출력했다.
- 독립 재검증: PNG 64장, 모든 크기 64×64, 카메라 위치·회전·ortho scale과 키 라이트 위치·회전·energy가 v2와 동일, shape key 최소 Z=0, root 위치 `(0,0,0)`·스케일 `(1,1,1)`·fcurve 0을 확인했다. W 방향의 연속 프레임 변화 픽셀은 201~229개다.
- UnityProject·기존 정지 자산·v1/v2 원본·렌더는 수정하지 않았다. Unity 반입은 미수행이다.
