# 작업 패킷: 백혈구 회피 대응 경험 스케일링

## 목표

기존 백혈구 회피 미니게임에 `면역 대응 경험`을 연결해, 내부 대응을 반복할수록 다음 백혈구 회피가 소폭 강화되도록 한다.

## 루프 단계

내부 바이러스 미니게임:

1. 면역 경계도 100%에서 내부 바이러스 모드로 전환한다.
2. 바이러스를 조종해 백혈구를 피하고 변이 조각을 수집한다.
3. 성공 또는 실패 결과가 다음 내부 대응의 난이도에 작은 흔적을 남긴다.

## 작업 단위

- 세션 상태에 면역 대응 경험 값을 추가한다.
- 첫 백혈구 회피는 기존 난이도를 유지한다.
- 두 번째 내부 대응부터 백혈구 속도 배율을 단계적으로 올린다.
- 실패한 내부 대응은 추가 경험을 남겨 다음 대응 강화폭을 조금 키운다.
- 기존 백혈구 접촉 포착값, 성공/실패, 변이 복귀 규칙은 유지한다.

## 비범위

- 백혈구 수 추가
- 새 스폰 패턴, 등장 간격, 씬 배치 변경
- 새 UI 패널 또는 에셋 추가
- 기본 내부 미니게임 타입 변경
- 면역 신호 억제 미니게임 규칙 변경

## 수용 기준

1. 첫 백혈구 회피 진입의 백혈구 속도 배율은 `1.0`이다.
2. 내부 미니게임 진입 경험이 누적되면 두 번째 이후 백혈구 속도 배율이 `1.0`보다 커진다.
3. 실패한 내부 미니게임은 추가 경험을 남겨 다음 백혈구 회피 속도 배율을 더 높인다.
4. 속도 배율은 설정된 최대값을 넘지 않는다.
5. 백혈구 접촉 `면역 포착 +8`, 성공 복귀 경계도 `25/33`, 실패 복귀 경계도 `60` 규칙은 유지된다.
6. Unity Test Runner EditMode와 MCP Play/Console 체크에서 회귀가 없어야 한다.

## 직접 구현 예외

- 사용자 요청은 승인된 쥐 숙주 1차 프로토타입 안의 기존 백혈구 회피 일부 수정이다.
- 실제 별도 하위 에이전트 실행은 생략하고, 역할별 수행 기록을 `agent-activity.md`에 남긴다.
- 새 씬, 패키지, 에셋, ProjectSettings 변경은 하지 않는다.
