# 핸드오프

## 최신 사용자 요청

v4보다 의도된 도트 게임 품질에 가까운 v5 픽셀 처리 시험을 진행한다.

## 현재 상태

- 상태: v5a 저밀도 진단안 보류, v5b 고밀도 픽셀 처리·시각/쿼터뷰 카메라 픽셀 스냅·960×540 정수 배율 Unity 통합 및 독립 QA 회귀 범위 통과. 사용자 화면 수용·총괄 판정 대기
- 유지: v4의 128×128/PPU64 및 2-unit 월드 폭
- 시험: 최대 32색, 무안티앨리어싱, 이진 알파와 불투명 중간색 픽셀 군집, 960×540 Point 출력, UI 분리

## 다음 작업

1. 사용자가 1920×1080(또는 실제 목표 해상도) 게임 창에서 v5b 픽셀 군집·선명도·정지/이동 안정성을 확인한다.
2. 사용자 수용 시에만 v4 공통 기준을 v5b로 교체할지 결정한다. 수용 전에는 v4 기준을 유지한다.

## Blender v5b 반입 인계

- 상태: 완료 — v4 읽기 전용 기반 v5b Blender 원본·64 PNG·프레임 맵·설정·픽셀 통계 생성 및 검증.
- 반입 후보: `renders-v5b/` 64장. v1~v4 및 v5a는 절대 덮어쓰지 않는다.
- 정량 결과: 최종 PNG 64/64, RGBA 128×128, 세트 공통 32-entry 팔레트(실사용 27색, 프레임별 21~27색), 중간 알파 0, 무디더, 보수적 고립 저대비 픽셀 병합 851px.
- 보존 결과: 8방향×8프레임, f08→f01, 최소 발바닥 Z 0, root `(0,0,0)`·fcurve 0, v4 카메라·조명·보행 계약.
- Unity 계약: Sprite Single / PPU64 / Pivot `(0.5,0.25)` / Point / Mipmap Off / Uncompressed / Clamp, 2 Unity unit 폭. 상세 통계와 배열 순서는 `verification.md`의 “Unity 반입 인계”를 따른다.

## 게임플레이 구현 완료분

- `RatVisual` 전용 수평 월드 격자 스냅 코드와 EditMode 계약 테스트를 추가했다. 기본값은 비활성이며, 통합 담당이 비교안에서 명시적으로 켤 수 있다.
- 접지 Y와 `0.005` clearance는 보존하므로 루트 이동·충돌·접지를 흔들지 않는다.
- 이 옵션은 팔레트·알파 처리 정책과 독립적이다. Dave the Diver/Binding of Isaac 참고로 재정리할 v5b 비교안에도 재사용 가능하다.
- v5b 시험 통합은 완료했다. 공통 기준 교체와 다른 캐릭터·전경 오브젝트 확대 적용은 사용자 수용 전 중지한다.

## 쿼터뷰 출력 픽셀 스냅 완료분

- `PrototypeCameraController`의 기본 비활성 opt-in 옵션은 쿼터뷰 직교 카메라에만 적용된다.
- 내부 RT 높이 `540` 기준 간격은 `2 × orthographicSize / 540`이며, final camera position의 right/up 화면 축만 스냅한다. forward 깊이와 회전은 보존한다.
- 비활성·비직교·무효 높이/크기에는 원 위치 fallback이다. v5b 통합 담당은 `960×540` Point RT 연결 시에만 이 옵션을 명시적으로 켠다.
