# 프로젝트 총괄 검토 — 쥐 v5b 고밀도 픽셀 처리 시험

검토일: 2026-07-21 KST
작업 ID: `2026-07-21-rat-pixel-treatment-v5`

## 검토 대상

- 사용자 승인: 쥐 스프라이트의 의도적 픽셀 처리 시험. 참고는 Dave the Diver와 Binding of Isaac의 **그래픽 언어만**이며, 특정 자산·캐릭터·화면을 복제하지 않는다.
- 대조 기록: `task.md`, `agent-activity.md`, `verification.md`, `handoff.md`, `work-log.md`, `CURRENT.md`, `docs/project-handoff/current-task-board.md`.

## 판정

**사용자 결정 필요**

v5b는 승인된 쥐 숙주 프로토타입의 시각 시험 범위와 3D 게임플레이 공간 + 프리렌더드 8방향 스프라이트 방향에 부합한다. 독립 QA의 Unity Play 회귀 범위도 충족했다. 다만 공통 제작 기준을 바꿀지는 사용자 화면 수용이 필요한 시각 결정이므로, 이 판정은 v5b를 최종 기준으로 확정하거나 작업 완료·커밋을 허용하는 뜻이 아니다.

## 근거

- v5a의 16색 전면 이진 알파안은 진단 비교안으로 보류되어 있으며, v5b는 별도 경로의 `128×128` 64장·세트 공통 최대 32색(실사용 27색)·무디더·중간 알파 0으로 제작됐다.
- 투명 윤곽은 이진 알파로 잠그되, 계단 표현은 불투명 중간 명도 군집으로 처리했다. 이는 특정 참고작을 모사하지 않고 사용자가 허용한 고수준 그래픽 언어만 시험한 것이다.
- `960×540` Point RenderTexture, 1920×1080에서 2배 표시 계약, HUD 분리, RatVisual/쿼터뷰 카메라 출력의 opt-in 픽셀 스냅은 3D 이동·충돌·면역·변이 루프를 바꾸지 않는 표시 계층 한정 변경이다.
- v1~v4와 v5a는 덮어쓰거나 삭제하지 않았고, v4의 `128×128 / PPU64 / 2 Unity unit` 공통 기준은 사용자 수용 전까지 유지하도록 작업 배정서·현황판·핸드오프에 명시돼 있다.

## QA/검증 기록 확인

- 독립 QA는 원본/Unity PNG SHA 64/64, 128px·27색 이하·중간 알파 0·무디더, Importer 64/64, 8방향 `f01 → f04 → f01`, PPU64 시각 격자 오차 0, 접지 clearance `0.0050`, QuarterView 출력 스냅, RT/HUD 계층, Play 콘솔 Error/Warning 0을 기록했다.
- 카메라 추적의 화면 평면 스냅 잔차도 기록됐다. QA는 MCP Play를 수행했고, 본 검토자는 이를 재실행하지 않았다.
- 남은 QA 한계: 실제 장시간 WASD 사용자 체감, OS 최종 합성 단계의 1920×1080 2배 출력 및 비정수 출력 레터박스/필러박스, 전체 EditMode TestRunner. 인자 없는 강제 카메라 호출의 도구 유발 예외는 현재 통과 근거에 포함하지 않는다.

## 기록 정합성

- `handoff.md`에 남아 있던 독립 QA 대기·통합 중지 문구는 현재 QA/통합 기록과 충돌해, 이번 검토에서 실제 상태로 동기화했다.
- `CURRENT.md`와 현재 상태판은 v5b QA 통과·사용자 수용/총괄 검토 대기 상태로 일치한다.

## 사용자에게 확인할 항목

1. 실제 게임 창에서 정지·직선/대각선 이동·카메라 추적 중 쥐 윤곽이 흐려지거나 픽셀 군집이 흔들리지 않는지.
2. v5b의 27색 명도 단계와 이진 윤곽이 목표로 한 도트풍 3D 가독성에 충분히 가까운지. 특정 참고작과의 동일함이 아니라, 읽기 쉬운 실루엣과 의도적 픽셀 질감만 판단한다.
3. 만족하면 v5b를 이후 프리렌더 캐릭터·전경 게임플레이 오브젝트의 공통 제작 기준 후보로 승격할지. 만족하지 않으면 v4 기준을 그대로 유지하고 v5b는 비교안으로 보관한다.
