- 사용자 승인: 2026-07-21, v5의 무안티앨리어싱·제한 팔레트·알파 픽셀 정리·Unity 고정 픽셀 격자/정수 배율 시험
- 담당 에이전트: Blender 애니메이션 테크아트, 게임플레이 구현, Unity 씬/통합, QA/검증

## 목적

v4의 선명한 저폴리 렌더를 의도된 도트 게임 화면으로 전환할 수 있는지 시험한다. v4와 같은 128px/PPU64 월드 크기를 유지하되, 픽셀 단위 윤곽·제한 팔레트·정수 배율 화면을 결합해 사용자가 시각 품질을 비교한다.

## v5a 진단안 (보류)

- 처음의 `16색 이하 + 전면 이진 알파 + 480×270` 안은 과도한 저밀도 비교용 진단으로만 남긴다.
- 사용자가 데이브 더 다이브·아이작의 그래픽 언어를 참고 기준으로 명시했으므로, 이 안을 최종 또는 Unity 반입 대상으로 사용하지 않는다.

## 확정 시험 기준 — v5b 고밀도 픽셀 처리

- Blender v5: v4를 읽기 전용 입력으로 사용, 128×128 RGBA·8방향×8프레임·기존 카메라/조명/보행/피벗/접지 보존
- 픽셀 처리: 렌더러의 부드러운 안티앨리어싱은 제거한다. 투명 알파는 픽셀 단위 이진 경계로 정리하되, 계단 보정은 반투명 알파가 아니라 불투명 중간 명도 색상 군집으로 만든다. 공통 팔레트는 최대 32색(투명 제외), 디더링 없음으로 제한한다.
- 그래픽 목표: 특정 자산을 복제하지 않고 정교한 픽셀 군집·풍부한 명도 단계·읽기 쉬운 실루엣·강한 대상 대비를 우선한다.
- Unity Import: Sprite Single, PPU64, Pivot `(0.5,0.25)`, Point, 밉맵 끔, 무압축, Clamp
- Unity 표시 시험: 쿼터뷰 월드를 `960×540` Point RenderTexture로 렌더한 뒤 `1920×1080`에서 정수 2배로 표시한다. 다른 출력은 레터박스/필러박스로 처리하며 HUD는 별도 UI로 유지한다.
- 픽셀 스냅: 시각 자식과 쿼터뷰 카메라 출력 위치만 내부 RenderTexture의 픽셀 격자에 맞춰 표시하고 이동·충돌·루트 Transform을 바꾸지 않는다.

## 해야 할 일

1. Blender 담당이 v5 원본·64 PNG·프레임 맵·렌더 설정을 별도 경로에 제작·검증한다.
2. 게임플레이 구현 담당이 RatVisual과 쿼터뷰 카메라 출력 위치만 픽셀 격자에 스냅하는 최소 구현·테스트를 만든다.
3. Unity 씬/통합 담당이 v5b를 별도 경로로 반입·연결하고 960×540 정수 2배 시험 출력을 구성한다.
4. QA가 팔레트/알파/Importer/화면 크기/정지·이동/8방향/픽셀 격자/콘솔을 독립 Play에서 확인한다.
5. 사용자 시각 확인 뒤에만 공통 제작 기준을 v5로 교체하거나 유지한다.

## 완료 기준

1. v5b 64장은 128×128·32색 이하·무안티앨리어싱·이진 알파 경계를 충족하고, 불투명 중간 명도 색상 군집·루프·방향·접지·피벗을 보존한다.
2. Unity에서 v5 64/64 Import 기준, PPU64, 2-unit 월드 폭, 8방향 `f01→f04→f01`을 확인한다.
3. 960×540 Point 출력의 2배 표시에서 정지·이동·대각선·카메라 추적 중 흐림·서브픽셀 떨림을 QA가 기록한다.
4. 사용자 확인 전에는 v4 공통 기준을 v5로 교체하지 않는다.
