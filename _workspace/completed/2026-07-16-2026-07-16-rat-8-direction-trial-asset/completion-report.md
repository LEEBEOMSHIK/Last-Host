## QA/검증 에이전트 판정

`완료 가능`

- Python `3.11.2`에서 자동 검증과 독립 재렌더를 수행했고, 13개 대상 파일의 재실행 전후 SHA-256이 일치했다.
- 단일 3D 원본 528 vertices/924 triangles, 투명 RGBA `64×64` 방향 PNG 8장, `512×64` 시트, 공통 피벗 `(32, 48)`, 투명 제외 최대 7색을 확인했다.
- 시트 방향 순서 `S, SW, W, NW, N, NE, E, SE`, 셀 내 여백, 주둥이·꼬리 방향 판독과 프리뷰 로컬 참조를 확인했다.
- S/N의 네이티브 1배 판독성, 걷기 애니메이션, Unity 환경의 접지·조명·가림은 미검증 후속 항목으로 분리했다.

## 프로젝트 총괄 관리자 판정

**내부 승인 가능**

### 근거

1. 사용자는 “8방향 쥐 시험 에셋 작업부터 진행하고, 내가 해야하는 부분 있으면 알려줘”라고 요청해 쥐 시험 에셋 제작을 명시적으로 승인했다.
2. 작업 범위는 쥐 1종, imagegen 콘셉트 참고 1안, 단일 저폴리 3D 원본 1개, 정지 8방향 1세트로 한정됐다.
3. imagegen 결과는 실루엣·비율·팔레트 참고안이고, 방향 PNG는 이를 8방향으로 개별 생성한 결과가 아니다. `render_rat_8dir.py::build_rat()`의 단일 메시를 고정 카메라·조명·스케일에서 회전해 재현했다.
4. Unity Import, SpriteRenderer·Animator·방향 매핑, PPU, 최종 셀 크기, 걷기·공격 애니메이션, 바닥 그림자·깊이 정렬, 최종 아트 확정은 수행하거나 승인된 것으로 기록하지 않았다.
5. Unity 코드·씬·프리팹·ProjectSettings·패키지와 `Builds/` 변경은 0건이다. 따라서 Unity MCP 플레이 체크는 대상이 아니다.
6. `.codex/config.toml`, `_workspace/previews/`, 자연 경계도 엄격 검증 패킷은 기존 별도 상태로 유지됐으며 이번 작업이 변경하지 않았다.
7. QA의 형식·재현성·시각 검사 기록은 시험 플레이스홀더를 사용자 비교 화면에 올리기에 충분하다.
