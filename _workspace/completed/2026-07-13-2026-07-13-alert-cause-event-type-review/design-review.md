# 기획 검토: 원인 라벨 enum/event type 전환

## 요약

표시용 `feedbackLabel`과 내부 대응 선택용 원인 타입을 분리하는 것은 승인된 쥐 숙주 프로토타입의 핵심 루프를 바꾸지 않는 구조 정리다. 기존의 `면역 경계도 상승 → 100% 도달 → 내부 미니게임 → 변이 선택 → 쥐 숙주 복귀` 흐름, 두 기존 내부 대응 유형, 보상·실패 의미를 그대로 유지한다면 범위 안에 머문다.

다만 실제 코드 전환은 게임플레이 코드 작성·수정에 해당한다. AGENTS.md의 승인 게이트에 따라 구현 전에 사용자 승인이 필요하다. 이번 검토는 전환 여부와 최소 설계 조건만 제안하며, 새 원인·새 미니게임·새 보상 규칙을 확정하지 않는다.

## 현재 의미와 보존 기준

2026-07-10 완료 기록의 문자열 기반 매핑은 다음 의미를 이미 갖고 있다.

| 원인 계열 | 현재 우선 내부 대응 | 보존해야 할 의미 |
| --- | --- | --- |
| 무라벨·기본·오염·면역 포착·바이러스 흔적 | 백혈구 회피 | 기본 회피/수집 대표 루프와 오염·포착 압박의 직관을 유지한다. |
| 강제 조종·소음·조직 자극·면역 신호·경보 | 면역 신호 억제 | 숙주 조작 또는 신호성 노출이 면역 경보를 증폭한다는 인과를 유지한다. |
| F6 디버그 진입 | 면역 신호 억제 | 디버그 진입의 현재 검증 목적을 유지한다. |

`feedbackLabel`은 플레이어에게 왜 경계도가 올랐는지 설명하는 문구이고, 원인 타입은 어떤 내부 대응을 기본 선택할지 결정하는 게임플레이 분류다. 같은 원인 타입이 여러 표시 문구를 가질 수 있고, 표시 문구가 바뀌어도 내부 대응의 의미가 바뀌지 않아야 한다.

## 최소 전환 원칙

- 원인 타입은 기존 매핑을 표현하는 제한된 분류만 둔다. 예: `Default`, `Contamination`, `ImmuneDetection`, `ViralExposure`, `ForcedControl`, `NoiseOrTissueIrritation`, `ImmuneSignal`.
- 내부 대응 선택은 타입→기존 `WhiteBloodCellEvasion` 또는 `ImmuneSignalSuppression`의 한 방향 매핑만 담당한다.
- UI는 타입의 표시명을 임의로 노출하지 않고, 호출 지점이 제공한 `feedbackLabel` 또는 타입별 기본 문구를 사용한다. 표시 문구의 현지화·표현 변경이 선택 결과를 바꾸면 안 된다.
- 타입 미지정·알 수 없는 타입·직접 `EnterVirusMinigame()` 호출은 기존과 같이 설정된 기본값 `WhiteBloodCellEvasion`으로 안전하게 폴백한다.
- 한 번의 경계도 상승 이벤트에는 하나의 명시적 원인 타입을 전달한다. 복합 원인이 필요해져도 이번 전환에서 새 우선순위 규칙이나 새 미니게임을 추가하지 않고, 기존의 백혈구 회피 우선 의미를 보존한다.

## 범위 판정

이 전환은 현재 이미 구현된 원인별 선택의 문자열 의존성을 제거하는 작은 리팩터링 후보다. 새 숙주, 새 맵, 새 내부 미니게임, 새 UI 패널, 새 입력, 보상·실패 규칙 변경을 요구하지 않는다. 따라서 쥐 숙주 프로토타입의 검증 질문(면역 압박을 이해하고 내부 대응을 거쳐 변이 후 복귀하는가)을 보강할 수는 있어도 확장하지 않는다.

반대로 아래는 별도 기획 판단 또는 범위 확대 신호다.

- 문서의 후보 유형(잠복 유지, 세포 위장, 손상 복구, 변이 회수)을 실제 선택 대상으로 추가하는 일
- 새 원인별 전용 시각화, 새 장애물, 새 보상/실패 효과를 추가하는 일
- 원인 누적·복합 원인에 따라 새 우선순위 또는 난이도 규칙을 만드는 일
- 면역 경계도 원인 자체의 의미나 기존 원인→미니게임 연결을 바꾸는 일

## 승인 필요

- **이번 설계 검토:** 사용자 요청으로 진행 중이며, 문서상 확정 결정을 바꾸지 않는 검토이므로 추가 승인 불필요.
- **실제 enum/event type 코드 전환:** 사용자 승인 필요. 게임플레이 코드 수정이며, 기존 원인→미니게임 매핑·기본 폴백·표시 문구 보존을 수용 기준으로 명시해야 한다.
- **새 타입 또는 매핑 의미 변경:** 사용자 승인 필요. 플레이어가 면역 압박 원인을 이해하는 방식과 내부 미니게임 맥락에 영향을 준다.

## 구현 전 수용 기준 제안

1. 기존 문자열 문구를 바꾸어도 동일한 명시 원인 타입이면 같은 미니게임이 선택된다.
2. 무라벨·미지정 타입·직접 진입은 계속 백혈구 회피 기본값으로 들어간다.
3. 오염·면역 포착·바이러스 흔적은 백혈구 회피, 강제 조종·소음/조직 자극·면역 신호/경보는 면역 신호 억제를 유지한다.
4. F6 디버그 진입, 기존 성공·실패·변이 선택·복귀 흐름이 회귀하지 않는다.
5. 새 미니게임·UI·입력·ProjectSettings 변경 없이 EditMode 회귀 테스트와 Unity Play 확인을 수행한다.

## 결론

원인 타입 분리는 프로토타입 범위 안의 안정성 개선으로 적절하다. 그러나 현재 문자열 매핑은 이미 플레이어 경험의 인과를 규정하므로, 구현 단계에서는 이를 단순 기술 리팩터링으로 간주하지 말고 위 보존 기준을 고정한 사용자 승인 후 진행해야 한다.
