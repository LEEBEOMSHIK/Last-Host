# 검증 기록

## 검증 대상

쥐 숙주 프로토타입 카메라 전환을 `3인칭 -> 후방 쿼터뷰 -> 탑뷰 -> 3인칭`으로 동작하게 하고, 쿼터뷰가 캐릭터 앞쪽이 아닌 뒤쪽 축에서 보이도록 고정한다.

## 실행한 검증

- RED 확인: Unity 동적 컴파일에서 `PrototypeCameraMode.TopView` 참조가 `CS0117`로 실패함을 확인
- 컴파일 확인: `PrototypeCameraMode.TopView` 참조 Unity 동적 컴파일 성공
- EditMode 테스트 직접 실행: `RatHostPrototypeCoreTests` 공개 테스트 17개 명시 호출
- Play 검증: `RatHostPrototype.unity` Play 모드에서 카메라 모드 순환, 쿼터뷰 후방 축, 탑뷰 수직 축, 쥐 프레이밍 확인
- 콘솔 확인: Unity 콘솔 Error 0건
- 패치 형식 확인: `git diff --check`

## 결과

- RED: `PrototypeCameraMode.TopView` 미정의로 실패 확인
- EditMode 테스트: 17/17 통과
- Play 검증: 통과
  - 쿼터뷰 오프셋: `(0.00, 7.40, -6.40)`
  - 탑뷰 오프셋: `(0.00, 9.50, 0.00)`
- 콘솔 Error: 0건
- `git diff --check`: 오류 없음, CRLF 변환 경고만 있음

## 검증하지 못한 항목

- 실제 물리 키보드 `V` 키 입력을 MCP에서 직접 누르는 방식의 수동 검증은 수행하지 못함
- 대신 `PrototypeKeyboardInput_MapsCameraToggleKey` 테스트와 Play 모드의 `ToggleHostCameraMode()` 경로로 전환 순서를 검증함

## 남은 위험

- 탑뷰는 수직 카메라이므로 카메라 기준 전방 벡터가 평면 투영 시 0에 가까워진다. 현재 입력 코드는 이 경우 `Vector3.forward`로 fallback 하므로 조작 축은 유지된다.
- 최종 체감 각도는 사용자의 실제 플레이 확인이 필요하다.

## QA/검증 에이전트 판정

- 판정: 조건부 승인
- 확인한 항목:
  - 쿼터뷰 오프셋이 `x=0`, `z=-6.4`로 변경되어 캐릭터 앞쪽/측면이 아닌 후방 축 기준으로 고정됨
  - 카메라 전환 코드가 `3인칭 -> 쿼터뷰 -> 탑뷰 -> 3인칭` 순환으로 구현됨
  - 탑뷰는 `topViewOffset (0, 9.5, 0)`과 정사영 카메라로 구현되어 상단 수직 시야 조건을 만족함
  - 테스트가 전환 순서, 후방 쿼터뷰 축, 탑뷰 수직 시야를 직접 검증함
  - `.codex/config.toml`은 이번 작업 판정에서 제외함
- 조건:
  - QA 판정을 이 파일에 반영할 것
  - 프로젝트 총괄 관리자 판정을 완료할 것
- 남은 위험:
  - 쿼터뷰의 후방은 월드 조작축 기준 후방이며, 쥐의 실시간 회전 방향을 따라가는 후방은 아님. 기존 카메라 고정 요구와는 맞지만 체감은 실제 플레이 확인이 필요함
  - QA는 Unity Play 검증을 재실행하지 않고, 메인 검증 기록과 diff 기준으로 판정함
