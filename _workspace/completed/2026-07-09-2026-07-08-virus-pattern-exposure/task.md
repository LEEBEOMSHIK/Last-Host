# 작업 패킷: 바이러스 패턴 노출 연결

## 목표

내부 바이러스 미니게임에서 백혈구 접촉이 `바이러스 패턴 노출`로 인식되도록 연결하고, 성공 후 변이 선택으로 복귀할 때 면역 경계도 부담으로 남게 한다.

## 범위

- 내부 바이러스 미니게임의 백혈구 접촉을 `면역 포착` 피드백으로 표시한다.
- 면역 포착 누적값을 변이 성공 후 쥐 숙주 복귀 경계도에 더한다.
- 실패 후 복귀는 기존 `보상 없음 + 경계도 60% 복귀` 규칙을 유지한다.
- 문서에는 현재 구현값과 장기 후보값을 구분한다.

## 비범위

- 새 내부 미니게임 유형 추가
- 새 UI 패널 또는 새 게이지 추가
- 바깥 쥐 이동만으로 바이러스 패턴 노출 발생
- 변이 능력별 노출 규칙의 전체 구현
- 씬 구조, 프리팹, ProjectSettings 변경

## 경량 루프 적용 사유

- 쥐 숙주 1차 프로토타입 승인 범위 안의 내부 미니게임-면역 경계도 연결이다.
- 새 시스템, 새 에셋, 새 패키지, 씬 변경 없이 기존 상태 모델과 HUD 문구만 수정한다.
- 사용자 비용 우려가 있었으므로 실제 병렬 하위 에이전트 실행은 생략하고, 역할별 산출물과 검증 책임을 작업 기록에 분리한다.
- 메인 통합자의 직접 구현 예외는 사용자 지시(`설계/문서 + 수용 기준 정리부터 하고 구현 진행`)와 이 작업 패킷의 경량 루프 사유에 근거한다.

## 수용 기준

1. 내부 바이러스 미니게임에서 백혈구에 접촉하면 `면역 포착 +8` 피드백이 기록된다.
2. 백혈구 접촉 없이 변이 조각 3개를 모아 성공하면 기존처럼 변이 선택 후 쥐 숙주 복귀 경계도는 25%다.
3. 백혈구에 1회 접촉하고도 변이 조각 3개를 모아 성공하면 변이 선택 후 쥐 숙주 복귀 경계도는 33%다.
4. 변이 선택 화면에서는 누적 흔적을 `면역 포착 흔적 +8`처럼 확인할 수 있다.
5. 내부 바이러스 미니게임 실패 후 복귀하면 보상은 없고, 기존 경계도 60% 복귀 규칙을 유지하며 노출 흔적은 다음 시도에 남지 않는다.
6. 단순 쥐 이동, 단순 탐색, 배경 노출만으로는 바이러스 패턴 노출이 발생하지 않는다.
7. 이번 구현은 백혈구 접촉 연결만 포함하고, `바이러스 흔적`, `변이 흔적`, 강한 변이 능력 사용 노출은 문서 후보로만 남긴다.

## 검증 계획

- EditMode 테스트를 먼저 추가해 RED를 확인한다.
- 구현 후 `RatHostPrototypeCoreTests` 전체를 Unity MCP로 실행한다.
- Unity MCP Play 체크에서 내부 바이러스 모드 진입, 백혈구 접촉 피드백, 변이 선택 화면 흔적, 복귀 경계도 33%, 콘솔 오류 0건을 확인한다.
- `git diff --check`로 문서/코드 공백 오류를 확인한다.
