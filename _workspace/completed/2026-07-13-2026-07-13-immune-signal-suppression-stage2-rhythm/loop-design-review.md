# 게임플레이 루프 설계 검토: 면역 신호 억제 2단계 리듬

## 루프 단계

- 위치: 면역 경계도 100% 이후의 내부 대응 중 `면역 신호 억제` 진입 구간이다. 기본 백혈구 회피를 대체하지 않는다.
- 단계 기준: `내부 대응 경험`은 같은 프로토타입 세션에서 내부 미니게임에 **진입한 횟수**다. 첫 진입은 1단계, 두 번째 및 이후 진입은 2단계로 고정한다.
- 경험 집계: 미니게임 유형과 성공·실패 결과에 관계없이 진입 시 1회로 집계한다. 따라서 첫 내부 대응이 백혈구 회피였다면, 다음 `면역 신호 억제`는 2단계다. 입력 누락·재시작·같은 대응 안의 개별 신호는 경험으로 추가 집계하지 않는다.
- 세션 리셋: 새 프로토타입 세션/명시적 세션 초기화 시 경험을 0으로 되돌려 다음 진입이 다시 1단계가 된다. 한 번 진입한 대응 안에서는 성공·실패 전환까지 단계가 바뀌지 않는다.

## 작업 단위

- 1단계 보존: 기존 고정 간격, 단일 입력, 기존 판정 창·성공·실패·안정도 규칙을 그대로 사용한다. 빠른 신호 표시는 1단계에서 발생하지 않는다.
- 2단계 신호열: 동일한 초기 상태에서 항상 같은 순서가 나오는 결정적 간격표를 사용한다. 일반 간격과 기준 간격보다 짧은 빠른 간격을 모두 포함하되, 2키·엇박·가짜 신호·유지 입력은 추가하지 않는다.
- HUD: 현재 리듬 단계(`1단계`/`2단계`)와 다음 신호 상태(`일반`/`빠름`)를 입력 전에 읽을 수 있게 표시한다. `빠름`은 시각 속도/간격과 HUD 표기가 일치해야 한다.

## 수용 기준

- 첫 내부 대응의 `면역 신호 억제`는 신호열 전체에서 단일 고정 간격만 사용하며, 다음 신호 상태가 `일반`으로만 표시된다.
- 두 번째 내부 대응부터는 2단계가 선택되고, 한 신호열 안에 최소 두 종류의 간격과 최소 한 개의 `빠름` 신호가 있다.
- 같은 단계로 새 모델을 시작하면 간격·빠름 위치가 동일하다. 랜덤 시드나 프레임 시간에 따라 신호열이 달라지지 않는다.
- 빠른 신호도 기존 단일 입력과 기존 판정/점수/안정도 규칙만 사용한다. 빠름이라는 이유로 별도 실패 규칙이나 보상은 생기지 않는다.
- 성공은 기존 변이 선택, 실패는 기존 실패 UI 및 쥐 숙주 복귀로 이어지며, 내부 대응 경험은 두 결과 모두에서 다음 대응에 보존된다.
- HUD는 대응 시작·신호 진행·완료/복귀에서 이전 단계나 다음 신호 정보를 잔류 표시하지 않는다.

## 테스트 시나리오

1. 새 세션에서 첫 내부 대응을 `면역 신호 억제`로 시작한다. 1단계, 고정 간격, `일반` 표기만 확인한다.
2. 첫 내부 대응을 백혈구 회피로 끝낸 뒤 신호 억제로 진입한다. 2단계와 변칙 간격·`빠름` 표기를 확인한다.
3. 첫 신호 억제의 성공 및 실패 각각 후 다음 내부 대응을 시작한다. 두 경우 모두 2단계가 유지되고 기존 성공/실패 복귀 흐름이 변하지 않는지 확인한다.
4. 2단계 모델을 같은 초기 조건으로 두 번 시작해 간격 배열과 빠름 위치가 완전히 같은지 확인한다. 일반·빠름 신호 각각에 정확/빠름·늦음/누락 판정을 적용해 기존 안정도 변화 규칙이 같은지 확인한다.
5. 세션을 명시적으로 초기화한 뒤 다시 진입한다. 1단계와 HUD 초기 상태로 돌아오며, 종료/복귀 후 단계·다음 신호 텍스트가 남지 않는지 확인한다.

## 범위 초과 여부

- 없음. 승인된 쥐 숙주 내부 대응 난이도 2단계의 최소 확장이다.
- 사용자 결정 필요 없음. 전역 내부 대응 진입 횟수를 기준으로 하는 판단은 작업 패킷의 “첫 내부 대응/두 번째 이후 내부 대응” 완료 기준과 일치한다.
- 구현 시 기존 상태가 `면역 신호 억제` 전용 완료 횟수만 보관한다면, 세션 공용 내부 대응 경험값을 읽도록 연결해야 한다. 새 장기 성장/원인별 난이도 시스템을 만들지 않는다.
