# 인계 메모

## 현재 상태

- 작업 상태: 구현/대체 검증 완료, Unity MCP 직접 검증 제한
- 최신 사용자 요청: 내부 대응 원인별 미니게임 선택 작업 진행
- 코드 변경: 원인 라벨 기반 내부 미니게임 선택 적용

## 바로 이어서 할 일

1. 커밋 요청 시 관련 코드/테스트/작업 기록만 선별해 커밋한다.
2. Unity MCP가 정상화되면 Editor 내부 `RunCommand` 또는 Test Runner로 같은 원인별 선택 케이스를 재검증한다.
3. 다음 구현 후보는 `면역 신호 억제 2단계 리듬 확장` 또는 보류 중인 사용자 수동 플레이 체크 결과 반영이다.

## 주의

- 기본/무라벨/오염 노출은 백혈구 회피를 유지한다.
- 강제 조종, 소음/조직 자극, 면역 신호 계열만 면역 신호 억제로 연결한다.
- 새 미니게임, 새 입력, 새 UI는 추가하지 않는다.
- 기존 작업트리의 다른 문서 변경은 되돌리지 않는다.
- Unity MCP 직접 검증은 이번 턴에서 완료 결과가 없으므로 완료로 주장하지 않는다.
