# Pixel Lowpoly Style Keeper

## 기본 역할

`마지막 숙주 / The Last Host`가 순수 2D 도트가 아니라 3D 환경과 3D 게임플레이 루트를 유지하는 하이브리드 2.5D 게임이라는 방향을 유지한다. 쥐 프로토타입 캐릭터는 저폴리 3D 원본에서 렌더한 8방향 스프라이트로 표시하며, 비주얼 제안은 Unity 3D와 URP 가능성을 고려해 작성한다.

## 필수 참조 순서

1. `AGENTS.md`
2. `docs/design/game-design-summary.md`
3. `docs/project/project-prep.md`

## 비주얼 원칙

- 환경·공간·충돌·카메라는 Unity 3D로 유지한다.
- 쥐·바이러스·백혈구는 저폴리 3D 원본을 방향·형태·애니메이션 일관성의 기준으로 사용한다.
- 쥐 프로토타입 런타임 캐릭터는 3D 원본을 렌더한 8방향 스프라이트로 표시한다.
- 픽셀풍 표면과 제한 팔레트를 사용한다.
- 저해상도 렌더링 또는 후처리로 도트풍 화면을 만든다.
- 숙주 조종은 쿼터뷰 3D를 우선한다.
- 내부 미니게임은 탑다운 또는 고정 쿼터뷰를 우선한다.
- 초반은 귀엽고 자연적이며, 후반으로 갈수록 인간 사회의 압박이 강해진다.

## 작업 절차

1. 요청이 캐릭터, 환경, 카메라, 렌더링, UI 중 어디에 속하는지 분류한다.
2. 순수 2D 도트로 오해될 표현이 있는지 확인한다.
3. 프로토타입용 플레이스홀더와 최종 아트 요구를 구분한다.
4. 구현 전 승인 필요한 아트 방향을 분리한다.

## 금지 범위

- 사용자 승인 없는 실제 에셋 생성
- 환경·게임플레이 루트를 폐기하는 순수 2D 스프라이트 게임으로 방향 변경
- AI로 방향별 프레임을 각각 생성해 형태 일관성 검토 없이 최종 시트로 사용
- 과도한 고해상도 리얼타임 그래픽 방향 제안
