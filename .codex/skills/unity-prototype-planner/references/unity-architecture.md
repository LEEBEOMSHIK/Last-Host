# Unity 구조 기준

## 전제

- 엔진은 Unity 3D다.
- PC 우선 프로토타입을 목표로 한다.
- URP는 추천이지만 생성 전 사용자 승인이 필요하다.
- 도트풍 3D 렌더링을 고려한다.

## 시스템 경계

- `Core`: 게임 상태, 모드 전환, 공용 이벤트
- `Host`: 쥐 숙주 이동, 상호작용, 숙주 생명력
- `Immune`: 면역 경계도, 상승/감소 규칙
- `VirusMinigame`: 내부 바이러스 이동, 백혈구, 변이 조각
- `Mutations`: 변이 선택, 효과 적용
- `UI`: 게이지, 모드 표시, 변이 선택 화면

## 씬 후보

- `Bootstrap`: 공용 초기화
- `RatHostPrototype`: 하수도 쥐 숙주 플레이
- `VirusMinigamePrototype`: 내부 바이러스 미니게임

초기에는 씬을 과하게 나누지 말고 전환 구조가 명확해진 뒤 분리한다.

## 승인 전 확인

- Unity 버전
- URP 사용 여부
- Input System 사용 여부
- 테스트 프레임워크 사용 범위
- 플레이스홀더 에셋 범위
