# 면역 신호 억제 리듬 미니게임

## 목적

`면역 신호 억제`는 내부 바이러스 미니게임을 백혈구 회피 하나로 고정하지 않기 위한 두 번째 유형 후보다. 숙주 내부의 면역계가 경보 신호를 박자처럼 퍼뜨리고, 플레이어는 바이러스가 그 신호 흐름을 타이밍 입력으로 끊어내는 장면을 플레이한다.

이 유형은 전투보다 `면역 경보를 교란한다`는 감각을 우선한다. 실제 면역학의 인터페론/사이토카인 개념은 설계 근거로만 쓰고, 기본 HUD 문구에는 직접 노출하지 않는다.

## 현재 위치

- 1차 쥐 숙주 프로토타입의 기본 내부 미니게임은 `백혈구 회피 + 변이 조각 수집`이다.
- `면역 신호 억제`는 현재 기본 미니게임을 대체하지 않는다.
- 2026-07-08 기준, Unity 코드에는 1차 판정 모델과 세션 연결이 추가되었다.
- 구현된 범위는 단일 입력, 단일 박자, HUD 텍스트 피드백, 전용 신호 패널, 성공/실패 루프 연결이다.
- 테스트 진입은 Play 중 `F6` 키로 가능하다. 기본 면역 경계도 100% 진입은 여전히 백혈구 회피다.
- 전용 패널은 런타임 HUD가 생성하며 신호 트랙, 판정선, 움직이는 신호 마커를 표시한다.
- 2026-07-10 기준, 정확 판정 직전의 `신호 접근` 예고 상태와 마커/판정선 펄스 피드백이 추가되었다.
- 노트 프리팹, 사운드 싱크, 원인별 자동 선택은 아직 후속 작업이다.

## 플레이 형태

리듬형 타이밍 입력 미니게임으로 설계한다.

1. 내부 화면에 면역 신호 파동 또는 경보 노드가 일정 박자로 접근한다.
2. 플레이어는 판정 지점에 신호가 도달하는 순간 입력한다.
3. 정확한 입력은 신호를 끊고 억제 점수를 올린다.
4. 빠르거나 늦은 입력, 입력 누락은 경보를 증폭시킨다.
5. 목표 억제량을 채우면 성공하고, 실패 한계를 넘으면 실패한다.

초기 구현은 음악 게임처럼 완성된 사운드 싱크를 요구하지 않는다. 첫 프로토타입은 시각 박자와 짧은 효과음만으로 검증하고, 사운드 기반 판정은 후속 단계로 둔다.

후속 확장에서는 음악 또는 심박/경보음 계열 루프를 입히고, 신호가 항상 정박으로만 오지 않도록 변칙 박자를 섞는다. 이때 음악은 단순 배경음이 아니라 신호 예고와 긴장감을 만드는 단서로 사용한다.

## 발생 조건 후보

| 조건 | 의미 | 우선도 |
| --- | --- | --- |
| 인터페론/사이토카인 경보가 주된 원인일 때 | 숙주 내부 경보 신호가 커짐 | 높음 |
| 바이러스 패턴 노출이 누적됐을 때 | 면역계가 바이러스 흔적을 신호로 추적함 | 중간 |
| 강제 조종 또는 조종 흔적이 반복됐을 때 | 숙주 이상 행동이 내부 경보로 번짐 | 중간 |
| 내부 대응을 여러 번 겪은 뒤 | 면역계가 더 조직적인 대응을 시작함 | 낮음 |

1차 확장 구현 후보는 `면역 경계도 100% 진입 원인이 신호/경보 계열일 때 면역 신호 억제를 선택한다`로 둔다. 원인별 미니게임 선택 시스템이 없을 때는 개발자 테스트용 진입 버튼이나 설정값으로만 검증할 수 있다.

## 성공 조건

최소 프로토타입 기준:

- 제한된 신호 묶음 안에서 목표 억제량을 채우면 성공한다.
- 성공 시 변이 선택으로 이어질 수 있다.
- 성공해도 면역계는 일부 학습하므로, 다음 내부 대응이 약간 강화될 수 있다.

권장 초안 수치:

| 항목 | 초안 |
| --- | --- |
| 신호 수 | 12개 |
| 성공 필요 입력 | 8개 |
| 판정 단계 | `정확`, `늦음/빠름`, `실패` |
| 입력 방식 | 단일 키 또는 좌/우 2키 |

이 수치는 구현 시작점일 뿐이며, 실제 체감 난이도는 Play 검증 후 조정한다.

## 실패 조건

다음 중 하나가 발생하면 실패한다.

- 실패 입력 또는 누락이 허용치를 넘는다.
- 바이러스 안정도가 0이 된다.
- 경보 증폭량이 한계에 도달한다.

실패 결과:

- 변이 보상 없음
- 쥐 숙주 모드로 복귀
- 기존 실패 UI 사용 가능
- 면역계가 경보 신호를 더 강하게 학습했다는 기록을 남길 수 있음

## 보상과 디메리트

| 결과 | 보상 | 디메리트 |
| --- | --- | --- |
| 성공 | 변이 선택 가능, 경보 신호 억제 연출 | 다음 대응 소폭 강화 가능 |
| 부분 성공 | 후속 설계 후보. 1차 구현에서는 사용하지 않음 | 보상 축소 후보 |
| 실패 | 보상 없음 | 바이러스 안정도 손실, 다음 대응 강화 후보 |

`면역 신호 억제`는 공짜 보상 방이 아니다. 플레이어가 리듬 입력을 잘하면 경보를 끊어 보상을 얻지만, 실패하거나 반복 진입하면 면역계가 더 정교하게 반응해야 한다.

## 난이도 확장

난이도는 면역 경계도가 차는 도중 바로 올라가지 않는다. 내부 대응을 반복하거나 같은 원인으로 재진입할 때 다음 미니게임이 강화된다.

| 단계 | 변화 |
| --- | --- |
| 1단계 | 단일 박자, 단일 입력 |
| 2단계 | 박자 간격 변화, 빠른 신호 일부 추가 |
| 3단계 | 음악/경보음 싱크, 변칙 박자, 짧은 엇박 신호 추가 |
| 4단계 | 좌/우 2키 분리, 가짜 신호 일부 추가 |
| 5단계 후보 | 긴 신호 유지 입력, 연속 입력 패턴 |

첫 구현은 1단계만 목표로 한다. 2단계 이상은 기본 리듬 판정과 UI 피드백이 안정된 뒤 추가한다.

## UI와 피드백 문구

HUD와 결과 문구는 짧게 유지한다.

| 상황 | 문구 후보 |
| --- | --- |
| 미니게임 목표 | `면역 신호 억제` |
| 정확 입력 | `신호 차단` |
| 누락 또는 실패 | `경보 증폭` |
| 성공 | `경보 신호 억제 성공` |
| 실패 | `면역 경보 억제 실패` |

기본 HUD에는 `사이토카인`, `인터페론` 같은 용어를 직접 쓰지 않는다. 필요하면 설명 문서나 도감성 UI에서만 사용한다.

## 구현 단위 후보

최소 구현 단위는 다음처럼 나눈다.

1. 순수 판정 모델
   - 신호 타이밍 목록
   - 입력 시간
   - 판정 결과
   - 성공/실패 결과
   - 1차 구현 완료: `ImmuneSignalSuppressionModel`
2. UI/시각화
   - 신호 접근 표시
   - 판정 지점
   - 성공/실패 피드백
   - 1차 구현 완료: HUD 텍스트, 안정도 표시, 전용 신호 패널, 신호 마커 이동
   - 후속 확장 완료: 정확 판정 직전 접근 예고 문구, 마커/판정선 색상 변화, 마커 펄스
   - 후속 필요: 전용 노트 프리팹, 사운드 싱크
3. 모드 연결
   - 내부 미니게임 진입
   - 성공 시 변이 선택
   - 실패 시 실패 UI와 쥐 숙주 복귀
   - 1차 구현 완료: 명시 타입 진입과 성공/실패 전환
   - 후속 필요: 경계도 원인별 자동 선택
4. 검증
   - 판정 모델 EditMode 테스트
   - Unity Play 체크
   - 실패/성공 루프 확인
   - 1차 구현 완료: EditMode 테스트 추가, 수동 컴파일/스모크 검증
   - 후속 필요: Unity MCP 승인 후 Play 체크

## 수용 기준

- 기존 백혈구 회피 미니게임을 대체하지 않는다.
- `면역 신호 억제`는 리듬 입력 기반 미니게임으로 이해되어야 한다.
- 플레이어는 목표, 성공, 실패, 디메리트를 UI 문구로 이해할 수 있어야 한다.
- 플레이어는 신호 마커가 판정선으로 접근하는 시각 피드백을 볼 수 있어야 한다.
- 실패는 보상 없음과 다음 압박 후보로 연결되어야 한다.
- 1차 구현은 단일 입력, 단일 박자 기반으로 시작해야 한다.
- 사운드 싱크가 없어도 시각 박자로 테스트 가능해야 한다.
- 2키 입력, 가짜 신호, 긴 입력은 후속 확장으로 둔다.
