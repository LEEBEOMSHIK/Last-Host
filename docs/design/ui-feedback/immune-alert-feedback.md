# 면역 경계도 피드백 규칙

## 목적

면역 경계도는 플레이어가 원인을 이해하지 못하면 `이유 없이 벌을 받는 게이지`처럼 보일 수 있다. 따라서 경계도 피드백은 단순히 숫자를 보여주는 것이 아니라, `무엇이 숙주의 방어를 자극했는지`를 짧고 반복 가능하게 알려줘야 한다.

## 현재 프로토타입 상태

현재 HUD는 면역 경계도 수치와 상호작용 프롬프트를 보여준다. 다만 경계도가 오른 직후 원인과 상승량을 별도 문구로 보여주는 피드백은 아직 상세 설계 단계다.

현재 수치 기준:

- 소음 배관 상호작용: `+15`
- 시간 경과 기본 상승: 비활성
- 100% 도달: 내부 바이러스 모드 전환

## 기본 피드백 구성

경계도 상승 피드백은 세 요소로 구성한다.

| 요소 | 목적 | 예시 |
| --- | --- | --- |
| 원인 라벨 | 왜 올랐는지 설명 | `소음/조직 자극` |
| 변화량 | 얼마나 올랐는지 설명 | `+15` |
| 단계 반응 | 위험 단계가 바뀌었는지 설명 | 게이지 색상, 경고음, 전환 연출 |

## 권장 문구

문구는 짧고 게임적인 표현을 우선한다. 실제 의학 용어는 너무 자세히 쓰지 않는다.

| 상황 | 권장 문구 |
| --- | --- |
| 소음 배관 상호작용 | `소음/조직 자극 +15` |
| 오염 구역 체류 | `오염 노출 +8` |
| 독성 물웅덩이 접촉 | `오염 자극 +10` |
| 내부 세포 침투 반복 | `바이러스 흔적 +8` |
| 강한 변이 능력 사용 | `변이 흔적 +12` |
| 숙주 생명력 손상 | `숙주 손상 +12` |
| 숙주 본능을 거스르는 강제 조종 | `강제 조종 +8` |
| 급회전/돌진 반복 | `조종 흔적 +6` |
| 내부 면역 포착 | `면역 포착 +20` |
| 잠복 강화 적용 | `잠복으로 상승 억제` |

## 단계별 HUD 반응

| 경계도 구간 | HUD 반응 |
| --- | --- |
| 0-24 | 기본 게이지, 추가 경고 없음 |
| 25-49 | 게이지가 짧게 점멸 |
| 50-74 | 색상 변화와 짧은 경고음 |
| 75-99 | 맥박형 애니메이션, 전환 임박 표시 |
| 100 | 내부 바이러스 모드 전환 연출 |

## 피해야 할 방식

- 원인 없이 숫자만 오르게 하지 않는다.
- 너무 긴 설명 문장을 HUD에 띄우지 않는다.
- `RIG-I`, `MDA5`, `사이토카인` 같은 실제 면역학 용어를 기본 HUD에 직접 노출하지 않는다.
- 플레이어가 피할 수 없거나 예측할 수 없는 상승을 초반 프로토타입의 주된 압박으로 삼지 않는다.
- `더러운 곳`이라는 배경 설명만으로 상승시키지 말고, 오염 접촉, 체류, 상처 상태 같은 조건을 함께 보여준다.

## 구현 후보

다음 구현 작업에서는 경계도 변경 이벤트에 `원인`, `상승량`, `중요도`를 함께 넘기는 방식을 검토한다. 문서상 권장 구조는 다음과 같다.

| 필드 | 예시 | 용도 |
| --- | --- | --- |
| 원인 ID | `NoisyPipe` | 로컬라이징과 로그 |
| 표시 라벨 | `소음/조직 자극` | HUD 표시 |
| 변화량 | `+15` | 플레이어 피드백 |
| 단계 변화 여부 | `true/false` | 추가 경고 연출 |

이 구현 후보는 아직 승인된 코드 변경이 아니다. 현재 문서는 다음 작업에서 검토할 피드백 기준만 정의한다.
