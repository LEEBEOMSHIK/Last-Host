# 게임 기획 요약

## 정체성

- 한국어 제목: `마지막 숙주`
- 영어 제목: `The Last Host`
- 장르: 도트풍 저폴리 3D 원본 기반 하이브리드 2.5D 진화 생존 로그라이크
- 핵심 판타지: 귀엽게 의인화된 바이러스가 숙주를 감염하고, 적응하고, 변이하며, 전이와 지배와 공존 중 생존 방식을 선택한다.

## 핵심 콘셉트

플레이어는 약한 바이러스로 시작한다. 살아남기 위해 숙주를 옮겨 다니고, 현재 숙주를 조종하며, 면역체계의 압박에 대응하고, 이후 플레이 방향을 바꾸는 변이를 선택한다. 최종 목표는 단순한 감염이나 정복이 아니다. 게임은 생존이 끝없는 변이인지, 지배인지, 공존인지, 사멸인지, 수용인지 묻는다.

## 메시지

원본 초안은 인간을 자연의 절대적 통제자가 아니라 생태계 안의 약한 존재로 본다. 가장 강한 생존 방식은 반드시 지배가 아니다. 공존과 적응도 중요한 생존 방식으로 제시된다.

## 비주얼 방향

확정된 비주얼 방향은 3D 게임플레이 공간과 프리렌더드 캐릭터 스프라이트를 결합한 도트풍 하이브리드 2.5D다.

- 환경·공간·충돌·카메라는 Unity 3D로 구성
- 현재 승인된 쥐 프로토타입의 쥐·바이러스·백혈구는 저폴리 3D 원본에서 렌더한 8방향 스프라이트로 표시
- 캐릭터의 3D 원본은 방향·형태·애니메이션 일관성의 제작 기준으로 유지
- 픽셀풍 표면과 제한 팔레트
- 저해상도 렌더링 또는 후처리
- 귀엽지만 기묘한 바이러스 디자인
- 초반에는 자연적이고 귀여운 분위기
- 후반으로 갈수록 인간 사회, 병원, 연구소, 백신으로 인해 어두워지는 분위기

이 게임은 순수 2D 스프라이트 게임이 아니다. 캐릭터가 스프라이트로 표시되더라도 이동·충돌·깊이·가림·카메라와 환경은 3D 게임플레이 공간을 기준으로 한다.

이 표시 방식은 현재 쥐 프로토타입 범위에 승인되었다. 이후 숙주와 전체 캐릭터 체인에 확대 적용할지는 각 마일스톤에서 별도로 결정한다.

## 게임 구조

게임은 두 가지 주요 모드를 오간다.

### 숙주 조종 모드

플레이어는 현재 감염한 숙주를 3D 환경에서 조종한다. 현재 승인된 쥐 프로토타입에서는 숙주의 런타임 외형을 3D 원본에서 렌더한 방향별 스프라이트로 표시하지만, 이동과 상호작용은 3D 공간을 기준으로 한다. 이후 숙주의 표시 방식은 각 마일스톤 승인에 따른다. 각 숙주는 단순 이동 수단이 아니라 서로 다른 이동 방식, 접근 가능 구역, 위험 요소, 전이 기회를 가진 플레이어 캐릭터다.

예상 숙주:

- 벌레
- 모기
- 쥐
- 새
- 고양이
- 개
- 가축
- 인간

### 내부 바이러스 모드

면역 경계도가 임계치에 도달하면 플레이어는 숙주 내부의 바이러스를 직접 조종한다. 이 모드는 탑다운 또는 고정 쿼터뷰의 작은 아레나형 미니게임이다.

예상 행동:

- 백혈구 회피
- 항체 또는 위험 패턴 회피
- 변이 조각 수집
- 특정 세포 지점 침투
- 제한 시간 동안 생존

## 핵심 루프

1. 현재 숙주를 조종한다.
2. 다음 숙주 또는 생존 기회를 찾는다.
3. 전이 또는 생존 조건을 충족한다.
4. 면역체계 반응이 발생한다.
5. 내부 바이러스 미니게임을 진행한다.
6. 변이 포인트 또는 변이 조각을 얻는다.
7. 업그레이드를 선택한다.
8. 변경된 능력으로 숙주 조종 모드에 복귀한다.

## 성장 계열

바이러스의 성장은 `변이`와 `숙주 경험 특성`을 분리해 다룬다. 변이는 바이러스 자체가 내부 미니게임과 선택을 통해 얻는 생물학적 성장이고, 숙주 경험 특성은 이전 숙주의 생존 방식을 배워 다음 숙주에서 제한적으로 활용하는 장기 특성이다.

### 전이

새로운 숙주와 환경에 접근할 수 있게 한다.

### 변이

면역체계와 백신 대응력을 높인다.

### 조종

숙주 행동을 더 강하게 조작하고 지배 계열 루트를 연다.

### 공존

숙주 손상과 면역 압박을 낮추고, 장기 생존과 공존 엔딩 방향을 강화한다.

### 숙주 경험 특성

각 숙주는 실제 동물의 생태와 능력에서 가져온 3~4개의 고유 특성을 가진다. 바이러스는 숙주 경험을 통해 그중 일부를 학습할 수 있고, 다음 숙주에서 제한적으로 사용할 수 있다. 상세 기준은 `progression/host-experience-traits.md`를 따른다.

## 주요 게이지

- 숙주 생명력
- 면역 경계도
- 침식도
- 공존도
- 인간 위험도
- 백신 개발도

첫 프로토타입에 모든 게이지가 필요하지는 않다. 1차 검증에서는 숙주 생명력, 면역 경계도, 변이 관련 값 1개나 2개에 집중하는 편이 적절하다.

## 스테이지 진행

원본 초안은 자연 생태계에서 인간 통제 공간으로 이동하는 진행을 제안한다.

1. 숲 바닥
2. 연못과 풀숲
3. 농장
4. 도시 하수도
5. 주택가
6. 병원
7. 연구소

첫 프로토타입은 도시 하수도 단계의 쥐 숙주에서 시작한다. 이 결정은 정식 진행 순서가 쥐부터 시작한다는 뜻이 아니다. 쥐는 벌레보다 상위 숙주에 가깝지만, 전체 초반 숙주 체인을 만들지 않고도 숙주 조종, 인간 생활권 접근, 내부 면역 반응을 함께 검증하기 좋은 수직 슬라이스다.

## 엔딩 방향

원본 초안은 다섯 가지 엔딩 방향을 정의한다.

- 슈퍼바이러스
- 지배
- 공존
- 사멸
- 수용

엔딩은 장기 설계 목표이며, 첫 프로토타입 구현 범위에는 포함하지 않는다.

## 카메라와 조작

- 숙주 조종 모드: 쿼터뷰 3D를 추천한다.
- 내부 바이러스 모드: 탑다운 또는 고정 쿼터뷰 아레나를 추천한다.
- 두 모드의 기본 조작감은 최대한 일관되게 유지하되, 바이러스 모드는 더 짧고 긴장감 있는 회피와 수집 중심으로 설계한다.

## UI 방향

전체 게임에서는 숙주 생명력, 면역 경계도, 침식도, 공존도, 변이 슬롯, 다음 숙주 후보, 인간 위험도, 백신 개발도를 표시할 수 있다. 첫 프로토타입에서는 핵심 루프 이해에 필요한 UI만 남긴다.

1차 프로토타입 권장 UI:

- 숙주 생명력
- 면역 경계도
- 현재 모드
- 미니게임 중 변이 조각 수
- 성공 후 변이 선택 화면

## 주요 미확정 질문

- 내부 미니게임의 정확한 성공 조건을 무엇으로 둘지
- 첫 프로토타입에서 변이 선택지를 몇 개 제공할지
- 쥐 프로토타입에서 침식도가 어느 정도의 조종 능력을 열어줄지
- 공존도를 첫 프로토타입에 기계적으로 반영할지, 다음 마일스톤으로 넘길지
