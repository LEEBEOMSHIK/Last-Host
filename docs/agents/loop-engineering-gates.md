# 루프 엔지니어링 게이트

최종 수정일: 2026-07-01

## 목적

루프 엔지니어링 중 메인 에이전트가 구현, 검증, 승인 판정을 혼자 처리해 완료로 보고하는 일을 막는다. 이 문서는 작업 완료와 커밋 전에 반드시 확인해야 하는 차단 게이트를 정의한다.

## 적용 대상

다음 작업은 이 문서의 완료 전 게이트를 적용한다.

- Unity 코드, 씬, ProjectSettings 변경
- 승인된 기획 문서, 구현 계획, 운영 문서 변경
- 커밋을 만들거나 완료로 보고하는 작업
- 사용자에게 내부 승인 또는 검증 완료를 주장하는 작업

단순 질문 답변, 파일 위치 안내, 실행 결과 조회처럼 프로젝트 산출물을 바꾸지 않는 작업은 적용 대상이 아니다.

## 용어 구분

- 에이전트: `.agents/*-agent.md`에 정의된 역할을 맡아 작업 패킷과 산출물을 남기는 주체
- 스킬: `.codex/skills/*` 또는 Codex 스킬 지침. 절차 보조 도구이며 에이전트 산출물을 대체하지 않는다.
- 메인 에이전트: 현재 사용자와 직접 대화하며 도구를 실행하는 Codex 주체

스킬을 사용해 검증 명령을 실행했더라도, QA/검증 에이전트 작업 패킷과 산출물이 없으면 `QA/검증 에이전트가 검증했다`고 보고하지 않는다.

## 완료 전 필수 게이트

작업 완료, 완료 보고, 커밋 전에는 다음 네 가지가 모두 있어야 한다.

1. 작업 배정 게이트
   - `_workspace/active/<작업ID>/task.md` 또는 완료 폴더의 `task.md`에 작업 목적, 담당 에이전트, 입력 자료, 금지 범위, 완료 기준이 있어야 한다.
   - 담당 에이전트가 정의된 작업이면 조정자가 바로 최종 산출물을 만들지 않고 담당 에이전트를 명시한다.

2. 담당 산출물 게이트
   - 구현, 문서, 검증 등 실제 수행 주체와 산출물이 `work-log.md`, `handoff.md`, 또는 `artifacts/`에 남아야 한다.
   - 메인 에이전트가 직접 수행한 경우에도 `담당 수행 주체: Codex 메인 에이전트`라고 명시한다.
   - 어떤 에이전트가 어떤 일을 처리했는지는 작업 폴더의 `agent-activity.md`에 에이전트별 역할, 담당 업무, 산출물, 판정으로 남겨야 한다.
   - 다른 에이전트에게 위임한 작업은 위임 요청, 받은 에이전트, 결과, 산출물 위치를 `agent-activity.md`에 기록한다.

3. QA/검증 게이트
   - 코드, 씬, ProjectSettings, 승인 문서, 운영 문서 변경은 QA/검증 에이전트 기준으로 검증 기록을 남긴다.
   - 검증 기록에는 원래 증상 재현 여부, 실행한 테스트/Play 검증/콘솔 확인, 미검증 항목, 완료 판단이 있어야 한다.
   - 구현한 주체가 실행한 명령만 있으면 독립 검증으로 보지 않는다. 독립 검증이 불가능하면 그 사유를 문제 사안으로 보고한다.

4. 총괄 관리자 게이트
   - 사용자 최종 보고 전 프로젝트 총괄 관리자 에이전트가 방향, 범위, 승인 게이트, QA 기록 존재 여부를 확인한다.
   - 판정은 `내부 승인 가능`, `수정 필요`, `사용자 결정 필요`, `보류` 중 하나로 남긴다.
   - `내부 승인 가능`이 아니면 완료 또는 커밋 완료로 보고하지 않는다.

## 커밋 전 차단 조건

다음 중 하나라도 있으면 커밋하지 않고 사용자에게 문제 사안으로 보고한다.

- `_workspace` 작업 패킷이 없다.
- 담당 에이전트가 정의된 작업인데 담당 산출물이 없다.
- `agent-activity.md`가 없거나 에이전트별 수행 내용과 판정이 누락되어 있다.
- QA/검증 에이전트 기록 없이 검증 완료를 주장해야 한다.
- 총괄 관리자 판정이 없다.
- 승인 게이트 여부가 불명확하다.
- 검증 실패 또는 미검증 항목이 완료 판단에 직접 영향을 준다.

## 사용자 보고 규칙

최종 보고에는 사용자가 확인해야 하는 파일만 제시한다. 작업 이력 파일은 사용자가 요청했거나 판단 근거 확인이 필요한 경우에만 보조 링크로 제공한다.
