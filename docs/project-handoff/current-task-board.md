# 현재 작업 후보와 핸드오프 현황

최종 갱신: 2026-07-10

## 목적

이 문서는 최근 완료한 작업, 아직 닫히지 않은 검증 공백, 다음 작업 후보를 사용자가 한곳에서 확인하기 위한 현재 상태판이다.

세부 실행 로그는 `_workspace/active/`와 `_workspace/completed/`에 남긴다. 이 문서는 누적 이력 파일이 아니며, 다음 작업 발굴 시 현재 후보와 우선순위를 갱신하고 오래된 항목은 정리한다.

## 운영 기준

- 현재 후보와 우선순위는 최신 상태로 갱신한다.
- 최근 작업 요약은 3~5개 정도만 유지한다.
- 완료된 상세 이력은 `_workspace/completed/`를 참조한다.
- 진행 중 상세 이력은 `_workspace/active/`를 참조한다.
- 판단이 끝난 미결 항목은 결과만 짧게 남기고 제거한다.

## 현재 저장소 상태

- 최신 커밋: `3e25287 feat: expand internal immune response tuning`

- ProjectSettings 미결 항목: `APP_UI_EDITOR_ONLY` 추가 변경은 사용자 요청에 따라 되돌림 완료

## 다음 작업 후보

### 1. MCP 승인 복구 후 누락 검증 마무리

- 우선순위: 높음
- 이유: 최근 면역 신호 억제 접근 예고 작업에서 Unity MCP가 `Stop` 호출 시 승인 해제되어 F6 진입/HUD 펄스 자동 검증이 닫히지 않았다.
- 범위: 코드 변경 없음. Unity MCP 승인 복구, Play 상태 확인, F6 진입, 콘솔 Error/Warning 확인.
- 추천 라우팅: QA/검증 에이전트 -> 프로젝트 총괄 관리자 에이전트
- 시작 요청 문장: `MCP 승인 복구 후 누락 검증 마무리 진행해`

### 2. 전체 플레이어블 검증 재정리

- 우선순위: 높음
- 이유: 공식 플레이어블 완료 기준에는 Windows 빌드 실행본 직접 플레이와 16:9 해상도 UI/카메라 검증이 남아 있다.
- 범위: 빌드 실행본 성공 루프 1회, 실패 루프 1회, `1920x1080`, `1600x900`, `1366x768`, `1280x720` 확인.
- 추천 라우팅: QA/검증 에이전트 -> 프로젝트 총괄 관리자 에이전트
- 시작 요청 문장: `전체 플레이어블 검증 재정리 진행해`

### 3. 상호작용 식별성 작업 최종 완료 처리

- 우선순위: 중간
- 이유: 자동 검증과 내부 승인은 있으나 사용자 Game View 수동 확인이 남아 active 상태로 유지되고 있다.
- 범위: 사용자가 보정 후 상호작용 대상 식별과 `Space` 입력을 확인하면 완료 보고로 닫는다.
- 추천 라우팅: QA/검증 에이전트 -> 프로젝트 총괄 관리자 에이전트 -> 문서/릴리즈 에이전트
- 시작 요청 문장: `상호작용 식별성 작업 최종 완료 처리 진행해`

### 4. 내부 대응 원인별 미니게임 선택

- 우선순위: 중간
- 이유: `면역 신호 억제`는 구현됐지만 기본 진입은 여전히 백혈구 회피다. 문서상 후속 필요 항목은 경계도 원인별 자동 선택이다.
- 범위: 면역 포착/바이러스 흔적/강제 조종 흔적 등 원인에 따라 백혈구 회피 또는 면역 신호 억제 진입을 선택.
- 추천 라우팅: 게임플레이 루프 에이전트 -> 게임플레이 구현 에이전트 -> QA/검증 에이전트
- 시작 요청 문장: `내부 대응 원인별 미니게임 선택 작업 진행해`

### 5. 면역 신호 억제 2단계 리듬 확장

- 우선순위: 중간
- 이유: 현재는 단일 박자 기반이다. 문서상 2단계는 박자 간격 변화와 빠른 신호 일부 추가다.
- 범위: 사운드/프리팹 없이 코드 기반 변칙 간격과 HUD 검증을 먼저 구현.
- 추천 라우팅: 게임플레이 구현 에이전트 -> QA/검증 에이전트
- 시작 요청 문장: `면역 신호 억제 2단계 리듬 확장 진행해`

## 최근 판단 항목

### ProjectSettings `APP_UI_EDITOR_ONLY`

- 상태: 되돌림 완료
- 변경 내용: `Standalone` scripting define에 `APP_UI_EDITOR_ONLY`가 추가되어 있었다.
- 의미: Unity `com.unity.dt.app-ui` 패키지의 Editor Only 설정으로, App UI 런타임 어셈블리를 플레이어 빌드에서 제외하는 define이다.
- 판단: 현재 프로토타입은 `UnityEngine.UI` 기반이고 App UI 런타임을 직접 쓰지 않지만, 사용자가 되돌림을 요청했으므로 ProjectSettings 변경은 커밋하지 않고 되돌렸다.

3. 그 다음 원인별 미니게임 선택 또는 신호 억제 2단계 확장으로 넘어간다.
