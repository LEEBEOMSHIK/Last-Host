# 쥐 숙주 프로토타입 승인 패킷

최종 수정일: 2026-06-29

## 목적

이 문서는 `쥐 숙주 프로토타입` 구현 전에 사용자 승인이 필요한 결정을 한곳에 모은다. 승인 전에는 코드 작성, 씬 생성, 에셋 생성, Unity ProjectSettings 변경을 진행하지 않는다.

## 현재 확정된 방향

- 게임은 Unity 3D 프로젝트로 진행한다.
- 첫 검증 목표는 `쥐 숙주 프로토타입`이다.
- 비주얼 방향은 순수 2D가 아니라 도트풍 저폴리 3D다.
- 프로토타입은 숙주 조종, 면역 경계도, 내부 바이러스 미니게임, 변이 선택 후 복귀의 최소 루프를 검증한다.
- 쥐는 벌레보다 상위 숙주지만, 첫 프로토타입에서는 정식 진행 순서가 아니라 핵심 루프 검증용 수직 슬라이스로 사용한다.
- 1차 프로토타입에는 벌레 튜토리얼을 포함하지 않는다.
- Unity 프로젝트 루트는 `UnityProject/`다.
- Unity 프로젝트 표시 이름은 현재 `My project`를 유지한다.
- Unity MCP는 읽기 호출이 가능한 상태이며, MCP를 통한 변경 작업은 별도 승인 대상이다.

## 승인 요청 항목

### 1. Unity 버전

- 현재 상태: `6000.4.6f1`
- 추천 결정: 1차 프로토타입은 현재 버전으로 고정한다.
- 이유: 이미 프로젝트가 해당 버전으로 생성되어 있고, 버전 변경은 ProjectSettings와 패키지 재검증 비용을 만든다.

### 2. URP 사용

- 현재 상태: URP 패키지와 렌더 파이프라인 설정 에셋이 포함되어 있다.
- 추천 결정: 1차 프로토타입은 URP를 사용한다.
- 이유: 도트풍 저폴리 3D의 저해상도 렌더링, 후처리, 조명 조정 실험에 유리하다.

### 3. 플랫폼

- 현재 방향: PC 우선
- 추천 결정: 1차 프로토타입은 PC 키보드 조작만 검증한다.
- 이유: 모바일 입력과 화면 배치는 핵심 루프 검증 이후로 미뤄도 된다.

### 4. 시작 범위

- 확정 결정: 쥐 숙주부터 바로 시작한다.
- 해석: 쥐는 벌레보다 상위 숙주지만, 첫 프로토타입은 정식 진행 순서가 아니라 하수도 쥐 구간만 잘라 핵심 루프를 보는 수직 슬라이스다.
- 제외: 벌레 튜토리얼은 1차 프로토타입에 넣지 않는다.
- 이유: 현재 프로토타입 목표는 숙주 조종, 면역 전환, 변이 복귀 루프 검증이며, 벌레 튜토리얼은 범위를 늘린다.

### 5. 초기 조작 방식

- 추천 결정:
  - 이동: `WASD`
  - 상호작용: `Space`
  - 카메라: 고정 쿼터뷰
  - 마우스 조작: 1차 프로토타입에서는 제외
- 이유: 숙주 모드와 바이러스 모드의 조작을 단순하게 맞춰 루프 이해를 우선한다.

### 6. 성공과 실패 조건

- 추천 성공 조건:
  - 쥐 모드에서 면역 경계도 100%에 도달하면 내부 바이러스 미니게임으로 전환한다.
  - 바이러스 미니게임에서 바이러스 안정도가 0이 되기 전에 변이 조각 3개를 수집하면 성공한다.
  - 성공 시 변이 1개를 선택하고 쥐 모드로 복귀한다.
- 추천 실패 조건:
  - 바이러스 안정도가 0이 되면 미니게임 실패로 처리한다.
  - 1차 프로토타입에서는 실패 화면과 재시도만 제공하고, 영구 사망이나 런 종료는 구현하지 않는다.
- 이유: 실패가 전체 루프 검증을 막지 않게 하면서도 내부 미니게임의 압박은 확인할 수 있다.

### 7. 초기 UI 게이지

- 추천 포함:
  - 숙주 생명력
  - 면역 경계도
  - 현재 모드
  - 바이러스 안정도
  - 변이 조각 수
- 추천 제외:
  - 침식도
  - 공존도
  - 인간 위험도
  - 백신 개발도
- 이유: 첫 빌드에서는 루프 이해에 직접 필요한 정보만 보여준다.

### 8. 프로토타입 변이 효과

| 변이 | 1차 프로토타입 임시 효과 |
| --- | --- |
| 잠복 강화 | 쥐 모드 면역 경계도 상승 속도 감소 |
| 신경 조종 | 쥐 이동 속도 또는 상호작용 속도 증가 |
| 포유류 적응 | 하수도 맵의 특정 통로 접근 또는 보상 지점 접근 허용 |

여기서 `1차 프로토타입 임시 효과`는 정식 다중 숙주 전이 효과가 아니다. 1차 빌드에 두 번째 숙주가 없기 때문에, `포유류 적응`을 골랐을 때 플레이어가 변화를 느낄 수 있도록 하수도 맵 안에서만 임시로 적용할 효과를 뜻한다.

### 9. 플레이스홀더 에셋

- 추천 결정: 1차 프로토타입은 기본 도형, 단순 저폴리 모델, 단색/픽셀풍 임시 머티리얼만 사용한다.
- 제외: 최종 캐릭터 모델, 완성형 애니메이션, AI 생성 아트 에셋
- 이유: 핵심 루프 검증 전에는 최종 아트 제작 비용을 쓰지 않는다.

### 10. 저장소와 Unity 메타 파일

- 현재 상태: Unity 프로젝트용 `.gitignore`가 있고, `.meta` 파일은 추적 대상이다.
- 추천 결정: 현재 정책을 유지한다.
- 이유: Unity 에셋 참조 안정성을 위해 `.meta` 파일은 저장소에 남겨야 한다.

## 범위 밖 항목

다음 항목은 1차 프로토타입 승인에 포함하지 않는다.

- 벌레 튜토리얼
- 여러 숙주 전이 체인
- 인간 숙주 플레이
- 병원, 연구소, 백신 시스템
- 엔딩
- 완성형 로그라이크 런 구조
- 영구 성장 시스템
- 최종 아트 에셋

## 승인 응답 형식

전체 추천안을 그대로 진행하려면 다음처럼 답한다.

```text
승인: 쥐 숙주 프로토타입 승인 패킷 전체 승인
```

일부만 바꾸려면 항목 번호와 변경 내용을 함께 답한다.

```text
수정 승인:
- 6번 실패 조건은 재시도가 아니라 쥐 모드 복귀로 변경
- 8번 포유류 적응 임시 효과는 임시 체력 증가로 변경
```
