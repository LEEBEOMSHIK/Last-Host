# 쥐 숙주 AI 보조 아트 제작 작업 순서

## 목적과 문서 상태

이 문서는 `쥐 숙주 1차 프로토타입`에서 ChatGPT 등 이미지 생성 도구를 콘셉트·래스터 초안 보조로 사용할 때의 승인 순서와 역할을 정한다. 실제 이미지, 텍스처, 모델, Unity 에셋을 생성하거나 적용하지 않는 계획 문서이며, 현재 프로토타입의 플레이스홀더를 최종 아트로 선언하지 않는다.

비주얼 기준은 `../../design/visual/pixel-lowpoly-3d-production-guide.md`, 승인된 기능 범위는 `../official/rat-host-prototype.md`, 구현 경계는 `rat-host-implementation-plan.md`를 따른다.

## 범위와 제외

### 대상 자산 묶음

| 묶음 | AI 보조로 비교할 수 있는 래스터 초안 | 별도 3D 작업 |
| --- | --- | --- |
| 쥐 숙주 | 털·피부·의복 없는 표면 색상, 제한 팔레트, 방향·실루엣 콘셉트 | 저폴리 쥐 메시, UV, 리깅, 애니메이션, 콜라이더 |
| 하수도 환경 | 바닥·벽·녹·누수·배관의 픽셀 패턴, 심리스 타일 후보 | 배관·벽·바닥 모듈 메시, UV, 충돌, 머티리얼 |
| 내부 면역 미니게임 | 바이러스·백혈구·변이 조각과 HUD 상태 아이콘 초안 | 아바타·적 메시, 충돌, UI 레이아웃, Unity 연결 |

### 제외

- 인간·병원·연구소·백신·캠페인·엔딩 등 승인 범위 밖 콘텐츠
- 생성 이미지를 최종 3D 메시, UV, 리깅, 콜라이더, Import, URP 설정의 자동 대체물로 쓰는 일
- 사용자 승인 전 실제 이미지 생성, 신규 아트 에셋 추가, Unity 씬·머티리얼·URP 변경

## 역할 배정

| 역할 | 책임 |
| --- | --- |
| 사용자 | 자산 묶음별 실제 생성 범위, 스타일 브리프, 최종 후보, Unity 적용 착수 승인 |
| 비주얼/테크 아트 담당 | 제한 팔레트·실루엣·텍셀 밀도 기준을 브리프로 정리하고, 생성 후보의 스타일 적합성을 검토 |
| 이미지 생성 도구 | 승인된 브리프 안에서 콘셉트·패턴·아이콘의 래스터 후보를 만든다. 최종 품질 판단이나 Unity 적용을 수행하지 않는다. |
| 3D 모델·Unity 통합 담당 | 승인된 후보를 참고해 저폴리 메시·UV·머티리얼·Import·씬 적용을 별도 작업 패킷에서 수행 |
| QA/검증 담당 | 출처·기록·승인 경계, 저해상도 가독성, Unity 콘솔·플레이·해상도별 결과를 독립 대조 |
| 프로젝트 총괄 관리자 | 사용자 승인, QA 기록, 범위 준수 여부를 검토해 내부 완료 판단 |

## 7단계 작업 흐름

| 단계 | 진입 기준 | 담당 | 수행과 기록 | 다음 단계로 가는 기준 |
| --- | --- | --- | --- | --- |
| 1. 자산 묶음 승인 | 프로토타입 범위와 필요한 시각 목적이 명확함 | 사용자, 조정자 | 묶음 ID, 용도, 생성 수, 제외 대상, 예산을 작업 패킷에 기록 | 사용자가 해당 묶음의 실제 생성을 승인함 |
| 2. 스타일 브리프·자산 목록 | 묶음 승인이 있음 | 비주얼/테크 아트 담당 | 대상별 화면 크기, 카메라, 팔레트, 실루엣, 텍셀 밀도, 반복 타일 여부, 금지 요소를 작성 | 생성 도구에 줄 정보와 검토 기준이 충분함 |
| 3. 승인된 범위의 생성 | 승인된 브리프와 자산 목록이 있음 | 이미지 생성 담당 | 프롬프트, 도구·날짜, 입력 참조 출처, 변형 번호, 원본 파일 위치를 생성 로그에 기록 | 승인 수 안의 후보가 있고 권리·개인정보 위험이 없음 |
| 4. 사람 선별·정리 | 생성 로그와 후보가 있음 | 비주얼/테크 아트 담당, 사용자 | 후보 비교, 선택 이유, 손 편집·심리스 연결·팔레트·투명 영역 정리 여부, 폐기 후보를 기록 | 사용자가 선택 후보와 후속 3D 준비를 승인함 |
| 5. 저폴리 메시·UV·머티리얼 준비 | 선택된 래스터 초안과 3D 작업 승인 | 3D 모델 담당, 비주얼/테크 아트 담당 | 별도 작업 패킷에서 메시 실루엣, 폴리곤 밀도, UV 텍셀 정렬, 머티리얼 계획을 작성·제작 | 메시·UV·텍스처가 3D 화면에서 읽히고 적용 범위가 확인됨 |
| 6. Unity 통합 | 3D 산출물과 Unity 변경 승인 | Unity 통합 담당 | Import, 머티리얼, 프리팹·씬, 카메라·URP 영향, 파일 위치를 별도 패킷에 기록 | 컴파일·콘솔·기본 플레이 체크에 문제가 없음 |
| 7. 시각·플레이 QA와 수용 | Unity 통합 결과와 검증 계획이 있음 | QA/검증 담당, 사용자, 총괄 관리자 | 목표 해상도 정지/이동 화면, 픽셀 안정성, 가독성, 성능, 플레이 영향, 출처 기록을 대조 | QA 완료 가능 및 총괄 내부 승인 뒤 사용자가 최종 수용 여부를 판단 |

## 사용자 결정 시점

1. 각 자산 묶음의 실제 생성 시작 여부와 수량
2. 스타일 브리프의 팔레트·분위기·금지 요소와 외부 참조 사용 가능 여부
3. 선별 단계의 최종 후보와 손 편집 방향
4. 저폴리 메시·UV·Unity 적용을 시작할 별도 승인
5. 시각·플레이 QA 결과를 바탕으로 한 실제 적용 수용 여부

어느 단계의 승인이 다음 단계 전체를 자동 승인하지 않는다. 특히 1~4단계의 래스터 생성 승인은 Unity 프로젝트 변경, 신규 패키지, URP 설정, 범위 확장 승인이 아니다.

## 필수 기록물

| 기록 | 최소 내용 |
| --- | --- |
| 자산 묶음 승인 | 묶음 ID, 용도, 승인일, 생성 수, 제외 범위 |
| 스타일 브리프·일관성 시트 | 카메라·팔레트·실루엣·텍셀 밀도·금지 요소·공용 아이콘 규칙 |
| 생성 로그 | 프롬프트, 생성 도구·날짜, 참조 출처, 후보/선택 변형, 원본 위치 |
| 선별·정리 기록 | 선택 이유, 손 편집 내용, 심리스·투명·팔레트 점검, 폐기 사유 |
| 3D·Unity 작업 패킷 | 생성 원본과 최종 파일 위치, 메시·UV·머티리얼·Import·씬 변경, 승인·검증 결과 |
| QA 기록 | 해상도별 화면, 이동 중 안정성, 콘솔·플레이 결과, 남은 위험 |

## 수용 점검

- [ ] 사용자는 자산 묶음별 실제 생성을 먼저 승인했다.
- [ ] 생성 후보는 도트풍 저폴리 3D의 제한 팔레트·큰 실루엣·가독성 기준과 충돌하지 않는다.
- [ ] 제3자 상표·캐릭터·로고·특정 작가 모사, 비공개 자료·개인정보 입력을 피했고 출처·권리 메모가 있다.
- [ ] 생성 원본과 사람이 정리한 최종 파일의 위치가 작업 패킷에서 추적된다.
- [ ] 생성 래스터는 저폴리 메시·UV·리깅·콜라이더·Unity Import를 대체하지 않으며, 각 작업이 별도 승인·검증되었다.
- [ ] Unity 적용 후 목표 해상도에서 텍스처·실루엣·UI가 선명하고 이동 중 깜빡임·흐림이 수용 가능한지 확인했다.
- [ ] 새 콘텐츠·시스템·UI가 쥐 숙주 프로토타입 승인 범위를 넘지 않는다.

## 위험과 대응

| 위험 | 대응 |
| --- | --- |
| 생성 결과가 순수 2D 스프라이트 방향으로 흐름 | 3D 실루엣·메시·카메라 검토를 먼저 하고, 래스터는 표면·아이콘 초안으로만 사용 |
| 타일 경계·투명 가장자리·팔레트가 게임 화면에서 깨짐 | 사람 선별·편집과 Unity 적용 뒤의 정지·이동 화면 검증을 분리 |
| 외부 저작물·상표·개인정보 문제 | 입력·참조 출처를 기록하고 금지 요소를 브리프에 명시, 불확실한 후보는 폐기 |
| 생성 이미지와 실제 UV·텍셀 밀도가 맞지 않음 | 메시·UV 준비를 별도 단계로 두고, 최종 파일 위치와 편집 내용을 기록 |
| 한 묶음의 승인 범위를 넘는 생성·Unity 변경 | 묶음 ID별 승인과 작업 패킷을 분리하고, 다음 단계 전 사용자 승인 여부를 대조 |

## 다음 실행 요청의 형태

실제 아트 작업을 요청할 때는 최소한 다음을 명시한다: 자산 묶음, 화면에서의 용도, 원하는 팔레트·분위기, 생성할 후보 수, 외부 참조 사용 여부, 생성 후 진행할 단계. 이 문서만으로 실제 생성이나 Unity 적용을 시작하지 않는다.
