# 쥐 숙주 면역 경계도 작업 방향

작성일: 2026-07-02

## 문서 상태

이 문서는 `쥐 숙주 1차 프로토타입`에서 면역 경계도, 숙주 본능, 내부 미니게임을 앞으로 어떤 순서로 다듬을지 정리한 작업 방향 문서다.

이 문서는 `docs/prototype/plans/rat-host-implementation-plan.md`를 대체하지 않는다. 기존 공식 구현 계획은 유지하되, 면역 경계도 관련 다음 작업을 더 작은 루프로 나누기 위한 보조 계획이다.

## 핵심 방향

면역 경계도는 `시간이 지나서 자동으로 차는 게이지`보다 먼저 `플레이어가 어떤 위험 행동을 했는지 이해할 수 있는 원인 기반 게이지`로 다듬는다.

작업 방향은 다음 순서를 따른다.

1. 면역 경계도 상승 원인을 플레이어에게 보여준다.
2. 오염/조직 자극 같은 환경 위험을 작은 오브젝트로 검증한다.
3. 숙주 본능과 강제 조종 충돌을 설계 스파이크로 검증한다.
4. 바이러스 패턴 노출은 내부 미니게임과 변이 행동 쪽으로 연결한다.
5. 내부 미니게임은 백혈구 회피를 1차 대표 유형으로 유지하되, 장기적으로 원인별 대응 유형을 확장한다.

## 작업 순서

### 1. 면역 경계도 원인 피드백

목적:

- 플레이어가 왜 면역 경계도가 올랐는지 즉시 이해하게 한다.

작업 내용:

- 면역 경계도 상승 이벤트에 원인 라벨과 변화량을 붙이는 구조를 설계한다.
- 현재 소음 배관 상호작용부터 `소음/조직 자극 +15` 같은 피드백을 보여준다.
- HUD에는 긴 설명 대신 짧은 원인 문구와 수치만 표시한다.

수용 기준:

- 소음 배관을 조사하면 면역 경계도 수치가 오르고, 상승 원인과 변화량이 함께 표시된다.
- 대기만으로 경계도가 오르는 오해가 없어야 한다.
- 같은 구조를 오염 구역, 강제 조종, 바이러스 흔적에도 재사용할 수 있다.

담당 후보:

- 게임플레이 구현 에이전트: 경계도 이벤트 데이터 구조
- Unity 씬/통합 구현 에이전트: HUD 표시 연결
- QA/검증 에이전트: Play 검증과 콘솔 확인

### 2. 오염/조직 자극 위험 구역

목적:

- `더러운 곳에 가면 경계도가 오른다`는 느낌을 배경 자동 상승이 아니라 명확한 접촉/체류 위험으로 검증한다.

작업 내용:

- 하수도 안에 작은 오염 구역 1개를 둔다.
- 쥐가 오염원에 접촉하거나 일정 시간 머무르면 면역 경계도가 오른다.
- 피드백은 `오염 노출 +8` 또는 `오염 자극 +10`처럼 짧게 표시한다.

수용 기준:

- 오염 구역 바깥에서는 경계도가 오르지 않는다.
- 오염 구역에 들어갔거나 머물렀다는 사실이 시각적으로 구분된다.
- 상승 원인 피드백이 소음 배관과 같은 규칙으로 표시된다.

담당 후보:

- 게임플레이 루프 에이전트: 위험 구역 수용 기준
- Unity 씬/통합 구현 에이전트: 오염 구역 배치
- QA/검증 에이전트: 접촉/비접촉 조건 검증

### 3. 숙주 본능과 강제 조종 스파이크

목적:

- 플레이어 조작 자체가 벌칙이 되지 않으면서도, 숙주 본능을 거스르는 강제 조작이 의미를 갖게 한다.
- 조종 능력이 낮은 바이러스는 숙주를 완전히 직접 조작하지 못하고, 숙주의 자연 이동에 간섭하는 느낌을 검증한다.

작업 내용:

- 쥐가 입력 없이도 느리게 자동 이동하는 최소 본능 이동을 둔다.
- 플레이어 입력과 숙주 본능 이동을 바이러스 조종력으로 블렌딩한다.
- 첫 검증은 단순 본능 이동과 `소음/오염 회피 본능`을 대상으로 삼는다.
- 강제 조종은 별도 게이지로 만들지 말고, 1차에서는 면역 경계도 상승 원인 중 하나로만 처리한다.
- `신경 조종` 변이는 조종력을 높여 자유 조작에 가까운 체감을 준다.

수용 기준:

- 입력이 없을 때 쥐가 느리게 자동 이동한다.
- 조종력이 낮을 때 플레이어 입력은 숙주 본능 이동을 일부만 꺾는다.
- 조종력이 숙주 저항보다 높을 때는 플레이어 입력이 거의 그대로 반영된다.
- 본능과 충돌하지 않는 평범한 개입은 경계도가 오르지 않는다.
- 쥐가 피해야 할 위험 방향으로 반복 입력할 때만 `강제 조종` 원인이 붙는다.
- 낮은 조종력으로 본능과 반대 방향을 강하게 밀면 이동 효율 저하와 `강제 조종` 피드백이 발생한다.

담당 후보:

- 게임플레이 루프 에이전트: 본능/강제 조종 규칙 확정
- 게임플레이 구현 에이전트: 최소 상태 모델 실험
- QA/검증 에이전트: 조작감 저해 여부 확인

### 4. 바이러스 패턴 노출 연결

목적:

- 바이러스다운 행동이 면역 감시망에 흔적을 남긴다는 규칙을 내부 미니게임과 변이 선택에 연결한다.

작업 내용:

- 1차 구현에서는 바깥 쥐 이동만으로 바이러스 패턴 노출을 만들지 않는다.
- 현재 1차 구현은 내부 미니게임의 백혈구 접촉을 `내부 감시망에 포착됨`으로 보고 `면역 포착 +8`을 기록한다.
- 백혈구에 접촉하고도 변이 조각을 모아 성공하면, 변이 선택 후 쥐 숙주 복귀 경계도에 누적 포착값을 더한다.
- 실패 후 복귀는 기존 실패 규칙인 `보상 없음`, `성공 복귀보다 높은 경계도`를 유지하고, 포착 흔적은 다음 시도에 남기지 않는다.
- 내부 미니게임에서 세포 침투 반복, 감시망 접촉, 강한 변이 능력 사용 같은 추가 후보는 다음 설계 대상으로 둔다.
- 피드백 문구는 `바이러스 흔적`, `변이 흔적`, `면역 포착` 계열을 사용한다.

수용 기준:

- 바이러스 패턴 노출은 플레이어가 바이러스 능력을 강하게 쓰거나 내부 감시망에 직접 포착됐을 때 발생해야 한다.
- 단순 이동, 단순 탐색, 배경 노출과 혼동되지 않아야 한다.
- 백혈구 접촉 시 HUD 또는 목표 문구에서 `면역 포착 +8`을 확인할 수 있어야 한다.
- 백혈구 접촉 없이 성공한 경우 변이 선택 후 복귀 경계도는 기존 25%를 유지해야 한다.
- 백혈구에 1회 접촉하고 성공한 경우 변이 선택 후 복귀 경계도는 33%가 되어야 한다.
- 실패 후 복귀는 보상 없이 60% 경계도로 돌아가며, 포착 흔적은 초기화되어야 한다.
- 이번 1차 구현은 새 내부 미니게임 유형, 새 게이지, 바깥 쥐 이동 노출을 추가하지 않는다.

담당 후보:

- 게임플레이 루프 에이전트: 트리거 후보 정리
- 게임플레이 구현 에이전트: 백혈구 접촉-포착 흔적-복귀 경계도 연결
- QA/검증 에이전트: EditMode 테스트와 Unity MCP Play 체크
- 문서/릴리즈 에이전트: 설계 문서 최신화

### 5. 내부 미니게임 유형 확장

목적:

- 내부 미니게임을 백혈구 전투 하나로 고정하지 않고, 원인별 대응 장면으로 확장할 수 있게 한다.

작업 내용:

- 1차 프로토타입은 현재처럼 백혈구 회피와 변이 조각 수집을 유지한다.
- 다음 설계 후보는 `면역 신호 억제`로 선택한다.
- `면역 신호 억제`는 경보 신호를 리듬 입력으로 끊는 내부 미니게임 유형으로 구체화한다.
- `잠복 유지`, `세포 위장`, `손상 복구`는 후속 후보로 남긴다.
- 미니게임 난이도는 경계도 상승 중 즉시 오르는 게 아니라, 내부 대응을 반복할수록 다음 대응이 강화되는 방식으로 둔다.

수용 기준:

- 1차 구현 범위가 갑자기 여러 미니게임으로 확대되지 않는다.
- 기존 백혈구 회피 미니게임을 대체하지 않는다.
- `면역 신호 억제`는 설계 문서에서 발생 조건, 성공/실패, 보상/디메리트, 난이도 기준을 갖는다.
- 반복 내부 대응 강화 규칙은 플레이어가 원인을 이해할 수 있어야 한다.
- 다음 미니게임 유형은 별도 승인 후 구현한다.

담당 후보:

- 게임플레이 루프 에이전트: 유형 선택과 성공/실패 조건
- Unity 씬/통합 구현 에이전트: 선택된 유형의 씬 구성
- QA/검증 에이전트: 반복 진입과 난이도 변화 검증

## 우선순위

1순위는 `면역 경계도 원인 피드백`이다. 현재 플레이어 혼란이 가장 직접적으로 발생하는 지점이고, 이후 오염 구역, 강제 조종, 바이러스 흔적에도 같은 구조를 재사용할 수 있다.

2순위는 `오염/조직 자극 위험 구역`이다. 현재 소음 배관과 함께 면역 경계도 상승 원인을 두 가지 이상으로 늘려 플레이어가 시스템 규칙을 비교할 수 있게 한다.

3순위는 `숙주 본능과 강제 조종 스파이크`다. 이 작업은 조작감에 영향을 줄 수 있으므로 바로 큰 구현으로 들어가지 않고 작은 실험으로 검증한다.

4순위는 `바이러스 패턴 노출`과 `내부 미니게임 유형 확장`이다. 이 둘은 내부 미니게임 구조와 변이 능력 확장이 필요하므로 1차 조작/피드백 안정화 후 진행한다.

## 승인 게이트

다음은 별도 사용자 확인 후 진행한다.

- 강제 조종으로 긴 입력 잠금, 반동, 비틀거림 같은 추가 조작 방해를 넣는 작업
- 내부 미니게임을 백혈구 회피 외 유형으로 추가 구현하는 작업
- 시간 경과 기본 면역 경계도 상승을 다시 켜는 작업
- 조종 부하를 별도 게이지로 분리하는 작업

## 다음 작업 제안

다음 실제 구현 작업은 `면역 경계도 원인 피드백`으로 둔다.

작업 패킷에는 다음을 포함한다.

- 현재 소음 배관 상호작용에서 원인 라벨과 변화량 표시
- 면역 경계도 변경 이벤트 구조 설계
- HUD 짧은 피드백 표시
- EditMode 테스트와 Play 검증 기준
- QA/검증 에이전트와 총괄 관리자 판정 기록
