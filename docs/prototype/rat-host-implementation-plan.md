# 쥐 숙주 프로토타입 구현 계획

## 문서 상태

이 문서는 사용자가 승인한 `쥐 숙주 1차 프로토타입` 구현 계획이다. 상세 작업 초안은 `_workspace/completed/2026-06-30-2026-06-29-rat-host-plan-agent/artifacts/rat-host-implementation-plan-draft.md`에 있고, 이 문서는 구현자가 따라야 할 공식 요약 기준이다.

## 승인된 전제

- Unity 버전: `6000.4.6f1`
- 렌더 파이프라인: URP
- 플랫폼: PC 우선
- 입력: 키보드 중심
- 기본 이동: `WASD`
- 상호작용/확인: `Space`
- 카메라: 고정 쿼터뷰
- 씬: `RatHostPrototype.unity`
- 게임 전용 폴더: `UnityProject/Assets/_Project/`
- 입력 액션 자산 후보: `UnityProject/Assets/_Project/Settings/Input/RatHostPrototypeControls.inputactions`
- 기존 `Assets/Scenes/SampleScene.unity`는 삭제하지 않고, 프로토타입 빌드 대상에서는 제외하거나 뒤로 미룬다.
- Unity MCP를 통한 승인 범위 내 씬, 에셋, 코드, ProjectSettings, Build Settings 변경은 허용한다.

## 구현 범위

1차 프로토타입은 다음 루프만 구현한다.

1. 작은 하수도 맵에서 쥐 숙주를 조종한다.
2. 시간 경과 또는 위험 행동으로 면역 경계도가 오른다.
3. 면역 경계도 100%에서 내부 바이러스 미니게임으로 전환한다.
4. 바이러스를 조종해 백혈구를 피하고 변이 조각 3개를 수집한다.
5. 성공 시 변이 1개를 선택한다.
6. 선택한 변이가 적용된 상태로 쥐 숙주 모드에 복귀한다.

## 제외 범위

- 벌레 튜토리얼
- 여러 숙주 전이 체인
- 인간 숙주 플레이
- 병원, 연구소, 백신 시스템
- 엔딩
- 완성형 로그라이크 런 구조
- 영구 성장 시스템
- 최종 아트 에셋
- 모바일 입력과 모바일 UI

## 기능 단위

### 1. 쥐 숙주 조종

- 쥐를 `WASD`로 이동한다.
- 하수도 맵 안에서 벽과 위험 요소를 구분한다.
- 관심 지점은 `Space`로 상호작용한다.
- 독성 물웅덩이 같은 위험 요소는 숙주 생명력 또는 면역 경계도에 영향을 준다.

### 2. 면역 경계도

- 시간 경과로 면역 경계도가 천천히 오른다.
- 위험 행동은 면역 경계도를 추가로 올릴 수 있다.
- 면역 경계도 100% 도달 이벤트는 한 번만 발생한다.
- 100% 도달 시 쥐 모드에서 내부 바이러스 미니게임으로 전환한다.

### 3. 모드 전환

- 쥐 모드 루트와 내부 바이러스 모드 루트를 명확히 분리한다.
- 전환 중 이전 모드 입력, UI, 충돌이 남아 동작하지 않게 한다.
- 성공 후 변이 선택을 마치면 쥐 모드로 복귀한다.

### 4. 내부 바이러스 미니게임

- 바이러스 아바타를 키보드로 이동한다.
- 백혈구 적 1종을 피한다.
- 변이 조각 3개를 수집하면 성공한다.
- 바이러스 안정도가 0이 되면 실패한다.
- 실패 시 실패 패널과 재시도를 제공한다.

### 5. 재시도

재시도 시 초기화:

- 바이러스 안정도
- 변이 조각 수
- 수집된 조각 상태
- 백혈구 위치 또는 패턴 시작점
- 실패 패널 표시 상태

재시도 시 유지:

- 쥐 모드 진입 전 큰 진행 상태
- 프로토타입 세션 상태

성공과 실패가 같은 프레임에 동시에 발생하면 성공을 우선한다.

### 6. 변이 선택

성공 시 세 가지 변이 중 하나를 선택한다.

| 변이 | 효과 |
| --- | --- |
| 잠복 강화 | 쥐 모드 면역 경계도 상승 속도 감소 |
| 신경 조종 | 쥐 이동 속도 또는 상호작용 속도 증가 |
| 포유류 적응 | 하수도 맵의 특정 통로 접근 허용 |

`포유류 적응`은 정식 다중 숙주 전이 효과가 아니라 1차 프로토타입의 임시 체감 효과다.

## 검증 기준

내부 구현 완료 조건:

- Unity 에디터 컴파일 오류가 없다.
- 승인된 범위 안에서만 Unity 씬, 코드, 에셋, ProjectSettings, Build Settings가 변경되어 있다.
- 핵심 상태 로직에 대한 EditMode 테스트가 통과한다.
- 최소 PlayMode 테스트가 통과하거나, 자동화 제외 사유가 기록되어 있다.
- 성공 루프와 실패 재시도 루프를 수동으로 확인했다.

플레이어블 프로토타입 완료 조건:

- Windows PC 빌드가 성공한다.
- 빌드 실행본에서 쥐 모드 시작부터 변이 선택 후 쥐 모드 복귀까지 한 번 이상 완료할 수 있다.
- 실패 후 재시도 루프가 동작한다.
- UI와 카메라가 16:9 PC 해상도에서 핵심 플레이를 가리지 않는다.

확인 해상도:

- `1920x1080`
- `1600x900`
- `1366x768`
- `1280x720`
