# 쥐 숙주 프로토타입

## 목표

`쥐 숙주 프로토타입`은 `마지막 숙주 / The Last Host`의 첫 플레이어블 검증 목표다. 더 많은 숙주, 스테이지, 엔딩, 인간 사회 시스템을 만들기 전에 핵심 루프가 실제로 작동하는지 확인한다.

## 기획 위치

쥐는 바이러스 입장에서 벌레보다 상위 숙주에 가까운 단계다. 첫 프로토타입을 쥐로 잡는 것은 정식 초반 진행 순서를 쥐부터 시작하겠다는 의미가 아니라, 핵심 루프를 검증하기 위해 하수도 쥐 숙주 구간만 잘라 보는 수직 슬라이스다.

벌레 튜토리얼과 하위 숙주 단계는 장기 진행 구조 후보로 남긴다. 1차 프로토타입에서는 쥐 숙주 조종, 면역 경계도, 내부 바이러스 미니게임, 변이 선택 후 복귀만 검증한다.

## 비주얼 표현 경계

- 하수도 환경, 이동 공간, 충돌, 깊이, 가림, 카메라는 Unity 3D를 유지한다.
- 쥐·바이러스·백혈구의 제작 원본은 저폴리 3D 모델과 리그를 기준으로 한다.
- 런타임 캐릭터 외형은 3D 원본에서 동일한 카메라·조명·크기로 렌더한 8방향 스프라이트로 표시한다.
- 스프라이트는 3D 게임플레이 루트의 시각 자식이며, 프레임 교체가 콜라이더·이동·위험 판정·상호작용 위치를 바꾸지 않는다.
- 현재 3D 플레이스홀더는 실제 스프라이트 제작·Unity 반입이 별도 승인될 때까지 유지할 수 있다.
- 프레임 해상도, PPU, 동작별 프레임 수, 그림자와 깊이 정렬 방식은 후속 시각 검증에서 결정한다.

## 검증 질문

플레이어가 다음 흐름을 재미있고 이해하기 쉽게 받아들이는지 확인한다.

1. 숙주를 조종한다.
2. 면역 압박이 상승하는 것을 느낀다.
3. 내부 바이러스 미니게임으로 전환된다.
4. 변이를 획득한다.
5. 변이 효과가 적용된 상태로 숙주 조종에 복귀한다.

## 포함 범위

- 작은 하수도 맵
- 쥐 숙주
- 기본 쥐 이동
- 단순 위험 요소 또는 막힌 경로
- 면역 경계도 게이지
- 면역 경계도 100% 시 모드 전환
- 내부 바이러스 아레나
- 바이러스 이동
- 백혈구 적 1종
- 변이 조각 수집
- 성공과 실패 결과
- 세 가지 변이 선택 화면
- 변이 선택 후 쥐 숙주 모드 복귀

## 제외 범위

- 전체 스토리 캠페인
- 벌레 튜토리얼
- 여러 숙주를 거치는 전이 체인
- 인간 숙주 플레이
- 병원과 연구소 시스템
- 백신 개발 시스템
- 최종 엔딩
- 완성형 로그라이크 런 구조
- 영구 성장 시스템
- 여러 숙주를 거친 숙주 경험 특성 계승 구현
- 최종 아트 에셋

## 핵심 루프

1. 바이러스가 조종하는 쥐로 하수도에 등장한다.
2. 좁은 맵을 탐색한다.
3. 시간 경과 또는 위험 행동으로 면역 경계도가 상승한다.
4. 면역 경계도 100%에서 내부 바이러스 모드로 전환한다.
5. 내부 미니게임에서 생존하고 변이 조각을 수집한다.
6. 성공하면 세 가지 변이 선택지를 보여준다.
7. 선택한 변이를 적용한다.
8. 변경된 상태로 쥐 조종에 복귀한다.
9. 실패하면 실패 UI를 보여준 뒤, 변이 보상 없이 쥐 조종에 복귀한다.

## 쥐 숙주 모드

### 목적

쥐 모드는 숙주 조종이 메인 탐험 레이어로 성립하는지 검증한다.

### 최소 기계 요소

- 이동
- 방향 전환
- 간단한 관심 지점 상호작용
- 시간 경과에 따른 면역 경계도 상승
- 숙주 생명력과 면역 경계도 표시

### 가능한 프로토타입 위험 요소

- 독성 물웅덩이
- 인간의 손전등 시야
- 막힌 배관
- 소리를 내는 바닥

첫 검증 빌드에서는 위험 요소 1개나 2개만 있어도 충분하다.

## 내부 바이러스 모드

### 목적

바이러스 모드는 플레이어가 숙주 몸 밖의 탐험에서 잠시 벗어나 숙주 내부의 면역 압박에 대응하는 전환이 긴장감을 주는지 검증한다.

### 최소 기계 요소

- 바이러스 아바타 이동
- 백혈구 회피
- 변이 조각 수집
- 목표 수량 또는 제한 조건을 만족할 때까지 생존

### 권장 초기 성공 조건

바이러스 안정도가 0이 되기 전에 정해진 수의 변이 조각을 수집한다.

백혈구 접촉은 내부 감시망에 포착된 것으로 보고 `면역 포착 +8` 흔적을 남긴다. 이 흔적을 가진 채 성공하면 변이 선택 후 쥐 숙주 복귀 경계도에 누적값이 더해져, 보상을 얻더라도 내부에서 드러난 바이러스 패턴이 부담으로 남는다.

실패 시에는 변이 선택지를 제공하지 않는다. 플레이어는 실패 UI를 확인한 뒤 쥐 숙주 모드로 복귀하며, 면역 경계도는 성공 복귀보다 높은 값으로 남아 다음 내부 대응 압박을 유지한다.

실패 UI 제목은 임시 문구로 조정 가능하다. 현재 추천 문구는 `면역 반응 돌파 실패`이며, 후보 문구로 `면역체계 격파 실패`, `면역 저항 돌파 실패`, `면역 방어 돌파 실패`를 둔다.

## 변이 선택지

첫 프로토타입은 원본 초안의 세 가지 선택지를 사용한다.

| 변이 | 계열 | 프로토타입 효과 |
| --- | --- | --- |
| 잠복 강화 | 공존 / 변이 | 면역 경계도 상승 속도 감소 |
| 신경 조종 | 조종 | 쥐 조종력을 높여 자유 조작에 가까워지고 이동 속도도 향상 |
| 포유류 적응 | 전이 | 이후 숙주 확장을 준비하는 선택지. 1차 프로토타입에서는 정식 숙주 전이가 아니라 하수도 맵의 특정 통로 접근을 허용하는 임시 효과로 검증 |

세 번째 선택지는 아직 두 번째 숙주가 없으므로, 1차 프로토타입 안에서는 `특정 통로 접근`으로 체감 변화를 검증한다.

바이러스가 숙주 경험을 통해 배우는 별도 특성 구조는 `docs/design/progression/host-experience-traits.md`에서 장기 기획으로 정리한다. 현재 쥐 숙주 프로토타입의 세 가지 변이 선택지는 이 구조와 별개로 유지한다.

## 최소 게이지

- 숙주 생명력
- 면역 경계도
- 바이러스 안정도 또는 바이러스 체력
- 변이 조각 수

이후 단계로 넘길 수 있는 게이지:

- 침식도
- 공존도
- 인간 위험도
- 백신 개발도

## 프로토타입 성공 기준

테스터가 별도 설명 없이 다음 내용을 이해할 수 있으면 프로토타입은 성공으로 본다.

- 왜 면역 경계도가 오르는지
- 왜 바이러스 모드로 전환되는지
- 미니게임에서 무엇을 해야 하는지
- 어떤 변이를 얻었는지
- 변이가 쥐 숙주 플레이를 어떻게 바꿨는지

## 주요 위험

- 모드 전환이 긴장감이 아니라 흐름 끊김처럼 느껴질 수 있다.
- 쥐 조작이 숙주 특유의 재미 없이 평범하게 느껴질 수 있다.
- 숙주가 하나뿐이면 변이 선택이 추상적으로 느껴질 수 있다.
- 도트풍 3D는 렌더링 조정 전까지 의도된 스타일이 아니라 단순히 저품질처럼 보일 위험이 있다.
- 프리렌더드 캐릭터가 3D 환경의 조명·그림자·가림과 맞지 않으면 종이처럼 뜨거나 방향 경계에서 프레임이 튀어 보일 수 있다.

## 구현 승인 상태

상세 승인 이력은 `docs/prototype/approvals/rat-host-approval-packet.md`에 정리한다. 사용자는 2026-06-30에 1차 프로토타입 핵심 구현 항목을 승인했고, 2026-07-16에 3D 환경과 3D 원본 기반 8방향 캐릭터 스프라이트를 결합하는 비주얼 방향을 추가 승인했다.

승인된 핵심 항목:

1. Unity `6000.4.6f1`
2. URP 사용
3. PC 키보드 우선
4. `WASD`, `Space`, 고정 쿼터뷰 조작
5. 성공/실패 조건 초안
6. 초기 UI 게이지 구성
7. 세 가지 프로토타입 변이 효과
8. 플레이스홀더 에셋 사용
9. Unity MCP를 통한 승인 범위 내 씬, 에셋, 코드, ProjectSettings, Build Settings 변경
10. 3D 환경을 유지하고 저폴리 3D 원본 기반 8방향 캐릭터 스프라이트로 표시하는 비주얼 방향 (2026-07-16 추가 승인)
